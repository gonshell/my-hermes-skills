# stock-deep-analysis（公司股票深度分析 SKILL）

## 一、SKILL 的目的与边界

### 1.1 这个 SKILL 解决什么问题

对某只上市公司做**半年期**深度分析，从基础数据到估值结论，提供可决策的整合报告。

### 1.2 触发条件

**触发**（用户说以下内容时）：
- "分析 XX 公司股票"
- "XX 股票估值"
- "XX 半年走势"
- "XX 公司基本面"
- "XX 投资价值分析"
- "deep dive XX"

**不触发**：
- "XX 短线怎么看"（这是技术分析，不在本 SKILL 范围）
- "XX 行业怎么样"（这是行业分析，应使用其他 SKILL）
- "XX 政策解读"（这是政策分析）

### 1.3 适用范围

- **市场**：A 股、港股、美股中概股
- **时间窗口**：默认 6 个月（可调整为 3 个月、1 年）
- **公司类型**：制造、消费、科技、金融均可（核心方法通用）

## 二、核心方法论：5 阶段 + 1 反思层

```
输入：股票代码 + 时间窗口
    ↓
【阶段 1】基础数据采集（事实层）
    ↓
【阶段 2】时点走势分析（走势层）
    ↓
【阶段 3】涨跌动力分析（因果层）
    ↓
【阶段 4】护城河与竞争分析（战略层）
    ↓
【阶段 5】估值参考分析（估值层）
    ↓
【阶段 6】贯穿全程的反思（元层）
    ↓
【整合】最终报告（8 章 + 3 附录）
    ↓
【阶段 7】事后验证（用户发起时）

```

### 关键：阶段 7 是事后验证的强制流程

当用户发起"验证报告"或"今日股价"等时间敏感任务时，必须按以下顺序执行：

**Step 1：识别时间窗口**
- "今日股价" → 盘后 5+ 小时，A 股已收盘，可查当日收盘价
- "本周走势" → 不预测（承认 1 周时间窗口超出方法论能力）
- "报告验证" → 区分"事实层"（可验证）vs"预测层"（需 3-12 个月时间窗口）

**Step 2：实时数据获取（强制）**
- **优先 curl 直接调用 API**（非搜索引擎）
- A 股实时数据首选：东方财富 push2.eastmoney.com API
- 港股首选：腾讯 qt.gtimg.cn API
- 美股首选：Yahoo Finance API
- **搜索引擎搜索不适用于实时数据**（返回的是历史快照）

**Step 3：诚实声明局限**
- 1 天时间窗口无法验证长期预测
- 任何"目标价""上行 X%"都需要 3-12 个月时间窗口
- 短期波动是随机游走，**不是预测对象**

详见 `references/real-time-data.md` 和 `references/validation-checklist.md`。

## 三、阶段 1：基础数据采集

### 3.1 必采集数据

| 数据类别 | 具体内容 | 来源 |
|---|---|---|
| 财务数据 | 营收 / 归母净利 / EPS / 毛利率 / 净利率 / ROE | 财报（年报/季报） |
| 销量数据 | 月度销量（如果是制造/消费） | 公司产销快报 |
| 行业数据 | 同行业务规模、市占率 | 中客网 / 中汽协 / 中商网 |
| 政策环境 | 行业相关政策、补贴、监管 | 工信部 / 交通部 / 证监会 |
| 资金面 | 主力资金、机构持仓 | 交易所 / 同花顺 / 东方财富 |
| 同行对比 | 至少 2 家同行的关键财务数据 | 各公司公告 |
| 全球对比 | 全球巨头的关键数据（如适用） | 海外公司年报 / 行业报告 |

### 3.4 关键节点的"反推起点价"技巧

半年分析需要「T-180 日」或「T-N 月」等历史节点的收盘价，但 A 股历史日线数据**通常不直接给任意一天**（券商 F10 给的是近 30/90/250 日线、最高/最低、当日数据）。**当历史日线缺失时，可用以下反推法**：

| 公开数据 | 反推关系 | 示例（海光信息 2025-12-22 起点） |
|---|---|---|
| 大宗交易披露「成交价 X 元，较**当日**收盘价 Y 折价 3%」 | 当日收盘价 = X / (1 - 0.03) | 12-23 大宗折价 3% 成交 212.93 → 当日(12-22)收盘 ≈ 212.93/0.97 ≈ 219.52 元 |
| 涨跌停「今日涨停价 = 昨收 × 1.20（科创板）」 | 昨收 = 涨停价 / 1.20 | 反推前一日收盘 |
| 公告「上月末股东户数 X 户，较上上月末减少 Y 户」 | 阶段跌幅 ≈ 同比指数跌幅 | 校验走势 |
| 同花顺/雪球 52 周最高/最低 | 反推该极值所在周均价 | 用作半年最高/最低点的近似锚点 |

**反推法的可靠度**：「成交价 / (1 - 折价率)」的误差 < 0.5%（折价率是披露固定值）；「涨停价 / 1.20」是机械公式，零误差；股东户数反推只能给方向，**不能给精确价**。所有反推值必须在报告中标注「反推估算（可靠度：中）」，不能伪装为披露值。

### 3.5 数据时点对齐陷阱（自审要点）

**常见错误**：在不同来源的简析/快报/点评中找"同比"数据时，**直接搬过来用，但忽略了它们对应的基础期间可能不同**。

例如海光信息「应收账款 YoY +90.05%」出自 2026-04 发布的 2026Q1 简析，对应的基础期间是 **2025-03 末**，不是 2024-03 末。「毛利率 55.6% 同比减 9.14pct」对应的基础是 2025Q1 的 61.19%。如果报告里把 2026Q1 和 2025 全年比、或把不同季度的简析数字混用一张表，就会出现"基础期不一致"的隐性错误。

**自检规则**：
1. 同比/环比类指标必须**显式标注 base 期间**（如 "YoY 2025Q1" / "vs 2025-12-31"）。
2. 不同来源的简析数字进表前必须**先核对 base 期**——同环比文字描述相同，base 期可能差一个完整年度。
3. 财报实际值（如年报、一季报正文）优先级 > 简析/点评/快报。**简析数字是「估算/口径可能调整」的，不能当披露值**。
4. 当发现某个数据点 base 期不明确时，要么查原始公告确认，要么**标 "unclear, 待原始公告核对"**，不能含糊使用。

### 3.2 数据采集的 3 个强制规则

**规则 1：标注来源和时点**
每个数据点必须标注（来源、时点）。例如：
- "2025 营收 414.26 亿（来源：年报；时点：2026-03 披露）"

**规则 2：区分披露 vs 估算**
- **披露数据**：年报/季报直接给出的数据
- **估算数据**：通过其他数据推算的（如 EBITDA = 营业利润 + 折旧摊销）

**规则 3：标注可靠度**
| 可靠度 | 含义 |
|---|---|
| 高 | 年报/季报直接披露，经审计 |
| 中 | 公司公告、券商研报 |
| 中低 | 通过其他数据推算的估算 |
| 低 | 单一来源、未交叉验证、营销话术 |

### 3.3 阶段 1 质量门槛（必做检查）

- [ ] 所有数据标注了来源和时点
- [ ] 区分了披露 vs 估算
- [ ] 标注了可靠度（高/中/中低/低）
- [ ] 同行数据至少 2 家
- [ ] 至少 1 个数据来源是独立第三方（非公司自报）

## 五、阶段 3：涨跌动力分析

### 5.1 核心方法：三层叠加模型

任何股票走势都有**多层驱动力**，必须分别识别：

**上涨动力（看涨时）**：
- 故事层（长期叙事）
- 兑现层（业绩兑现）
- 催化层（事件触发）

**下跌动力（看跌时）**：
- 业绩兑现失败
- 叙事动摇
- 估值消化

### 5.2 关键操作

| 操作 | 输出 |
|---|---|
| 列出 3 个层次的驱动力 | 表格：层次/时点/数据/逻辑 |
| 标注每个驱动力的强度 | ★~★★★ |
| 找出反常断言 | "最反常识的发现" |

### 5.3 反常断言的识别标准

满足任一条件即为反常断言：
- 业绩下滑但毛利率/现金流改善（质量信号）
- 估值下跌但基本面未变化（情绪驱动）
- 行业整体下跌但该公司上涨（独立 alpha）
- 行业整体上涨但该公司下跌（独立风险）

### 5.4 阶段 3 质量门槛

- [ ] 上涨/下跌动力各分 3 个层次
- [ ] 每个层次有数据证据
- [ ] 标注了反常断言
- [ ] 区分了"故事层"vs"兑现层"vs"催化层"

## 六、阶段 4：护城河与竞争分析

### 6.1 双重视角

**视角 1：行业层面（波特五力）**
- 供应商议价
- 买方议价
- 新进入者威胁
- 替代品威胁
- 行业内竞争

**视角 2：公司层面（巴菲特 4 类护城河）**
- 成本优势
- 品牌资产
- 转换成本
- 网络效应

### 6.2 横向对比的强制规则

**核心原则**：任何"领先""超越""最强"等表述**必须给出对比基准**。

| 错误写法 | 正确写法 |
|---|---|
| "宇通研发投入领先金龙" | "宇通 17.88 亿 vs 金龙 8.23 亿，2.17 倍" |
| "比亚迪海外份额领先" | "比亚迪欧洲市占率 23.6%（具体国别）" |
| "公司技术护城河深" | "对比同行具体技术指标，差距 X%" |

### 6.3 4 道护城河分析模板

对每道护城河，按以下格式分析：

```
护城河：[成本优势 / 品牌资产 / 转换成本 / 网络效应]

核心问题：[什么是最难复制的]

客观证据：
- 数据点 1
- 数据点 2
- 数据点 3

强度评估：[强/中/弱]

可证伪测试：[什么情况下这道理会不成立？]
- 条件 1
- 条件 2

关键监控指标：[定期检查的指标]
```

### 6.4 阶段 4 质量门槛

- [ ] 行业层面和公司层面都有覆盖
- [ ] 至少 2 家同行对比
- [ ] 至少 1 家全球巨头对比（如适用）
- [ ] 每个护城河都有可证伪测试
- [ ] 关键监控指标明确

## 七、阶段 5：估值参考分析

### 7.1 估值方法适用性评估（必做）

**4 种主要估值方法的适用条件**：

| 方法 | 适用场景 | 不适用场景 |
|---|---|---|
| PE 法 | 业绩稳定 + 行业可比公司多 | 业绩波动大 + 没有可比公司 |
| DDM 法 | 高分红 + 增长稳定 + g < r | 高增长 + 分红不可持续 |
| EV/EBITDA 法 | 有可信 EBITDA + 可比公司 | EBITDA 是估算 + 没有可比 |
| DCF 法 | 稳定成熟企业 | 增长波动大 |

### 7.2 估值必须给区间，不给目标价

**核心原则**：任何股票估值都有 ±30% 的不确定性，**估值 ≠ 预测**。

```
错误：目标价 40 元
正确：估值参考区间 25-40 元（不确定性 ±30%）
```

### 7.3 情景分析模板

| 情景 | 2026E 净利 | 同比 | 假设 PE | 估值区间 | 概率（主观） |
|---|---|---|---|---|---|
| 牛 A | 65 亿 | +17% | 22 | 57-65 | 5% |
| 牛 B | 62 亿 | +12% | 18 | 45-50 | 15% |
| 基 A | 60 亿 | +8% | 15 | 36-41 | 25% |
| 基 B | 55 亿 | -1% | 13 | 29-33 | 30% |
| 熊 A | 50 亿 | -10% | 12 | 24-27 | 20% |
| 熊 B | 45 亿 | -19% | 10 | 18-21 | 5% |

**期望收益** = Σ(概率 × 收益)

### 7.4 估值的关键免责声明

必须明确说明：
- 样本不足（历史回测可能 < 10 个样本）
- 方法局限（多数方法不适用该公司）
- 数据估算（EBITDA 等是估算的）
- 概率主观（情景概率是叙事概率，不是统计概率）
- 预测不可能（任何"目标价"都是估值不是预测）

### 7.5 阶段 5 质量门槛

- [ ] 评估了每种方法的适用性
- [ ] 给出估值区间而非目标价
- [ ] 做了情景分析（6+ 情景）
- [ ] 明确不确定性（±30% 或具体数字）
- [ ] 列出了关键监控指标
- [ ] 包含重要免责声明

### 7.6 估值的"诚实声明"必须项（必做）

每次给出估值结论时，必须包含以下"诚实声明"块：

```
⚠️ 估值的诚实声明：
- 本报告不构成投资建议
- 估值 ≠ 预测：估值是合理价格区间，预测是未来价格走势
- 市场可能长期偏离估值区间（参考 2022 年 6.01 元低估用了 2 年修复）
- 任何"目标价 X 元"是估值不是预测
- 时间窗口限制：3-12 个月时间窗口才能验证估值
- 情景概率是叙事概率（narrative probability），不是统计概率
- ±30% 不确定性是经验估计，不是精确计算
```

**错误的表述（禁止）**：
- ❌ "目标价 40 元，上行空间 40-55%"
- ❌ "83% 准确率"
- ❌ "未来会涨"
- ❌ "买入，目标价 X 元"

**正确的表述（推荐）**：
- ✅ "估值参考区间 25-40 元（不确定性 ±30%）"
- ✅ "上行空间 20-40%，但需 3-12 个月时间窗口验证"
- ✅ "情景概率 30% 是主观判断，不是统计概率"
- ✅ "本估值不构成投资建议"

## 八、阶段 6：贯穿全程的反思（元层）

### 8.1 反思的 5 个核心问题

每个阶段结束都要问自己：

| 问题 | 检查 |
|---|---|
| 这个结论是基于**横向对比**还是孤立判断？ | ☐ |
| 是否做了**反证测试**？ | ☐ |
| 数据是**披露**还是**估算**？ | ☐ |
| 是否标注了**主观 vs 客观**？ | ☐ |
| 是否承认了**不确定性**？ | ☐ |

### 8.2 避免的常见错误

| 错误 | 防范 |
|---|---|
| 评分矩阵无标准（"9/10"） | 改用横向对比 + 相对位置 |
| 目标价精确数字（"40 元"） | 用区间（"25-40 元"） |
| 概率加权精确数字（"40.5 元"） | 用加权区间（"31-37 元"） |
| 样本不足下的"X% 准确率" | 明确说明"X 个样本无统计意义" |
| 营销话术作为事实（"行业首个"） | 标注"公司宣称""待第三方验证" |

### 8.3 整个 SKILL 的最终质量自检

完成后必须检查：
- [ ] 所有"领先""超越""最强"都有对比基准
- [ ] 所有核心结论都有反证条件
- [ ] 所有数据都有来源标注
- [ ] 所有主观判断都有明确标注
- [ ] 不确定性范围明确（不是单一数字）

## 九、阶段 7：事后验证（重要！本 session 关键教训）

### 9.1 触发条件

用户发起以下任务时启动阶段 7：
- "验证报告"
- "今日股价"
- "今天 X 涨/跌多少"
- "市场表现"
- 任何时间敏感的事实型问题

### 9.2 核心原则

**绝对不能做的事**：
- ❌ 搜不到数据时"估计"（如"估计 27-30 元"）
- ❌ 偷偷给一个看似精确的"估计值"继续后续分析
- ❌ 假装 API 失败但仍继续工作
- ❌ 让"估计"污染后续所有结论

**必须做的事**：
- ✅ 用 curl 直接调用 API 拉取实时数据
- ✅ 没有数据时明确说"搜不到"+列方案+等用户决策
- ✅ 时间窗口不足时明确说明（如"1 天无法验证 3-12 月预测"）
- ✅ 区分"事实层"（可验证）和"预测层"（需时间窗口）

### 9.3 实时数据获取的 API 端点（必记）

**A 股首选 — 东方财富 push2 API**：
```bash
# 实时行情
curl -s -A "Mozilla/5.0" \
```

### 9.4 事后验证的标准流程

```
Step 1：识别时间窗口
        - 用户问什么时点？当日？本周？上月？
Step 2：拉取实时数据（curl API）
        - 必须真实数据，不是估计
Step 3：对比报告预测
        - 报告预测：XX 元 / 上行 X%
        - 实际数据：YY 元 / 实际 Y%
        - 偏差：YY - XX = 偏差
Step 4：诚实声明局限
        - 时间窗口是否足够？
        - 预测类型：估值 / 短期价格 / 长期趋势
        - 该预测在什么时间窗口下有效？
```

### 9.5 验证报告的诚实声明模板

```
本验证的局限：
- 报告生成时点：2026-XX-XX
- 验证时点：2026-XX-XX
- 时间窗口：N 天
- 1 天时间窗口无法验证 3-12 个月时间窗口的预测
- 事实层（基本面数据）可验证 ✅
- 预测层（估值、上行空间、目标价）需更长窗口 ⚠️
```

### 9.6 常见错误（来自本 session 的真实教训）

**错误 1**：搜索引擎搜不到 → 估计 27-30 元
- **正确做法**：用 curl 拉腾讯 API 或东方财富 API，10 秒拿到真实数据

**错误 2**：拿到数据后"继续基于估计做后续验证"
- **正确做法**：基于真实数据做严谨验证，任何不符合事实的结论立即停止

**错误 3**：用 1 天数据"验证"长期预测
- **正确做法**：明确说明"1 天无法验证长期预测"+列时间窗口要求

**错误 4**：用户问"今日股价"后默默做主观预测
- **正确做法**：明确说"无法预测"+提供条件式分析框架

## 十二、SKILL 的常见错误防范清单

### 12.1 必须在写作时检查的错误

- ❌ "宇通研发投入领先金龙" → ✅ "宇通 17.88 亿 vs 金龙 8.23 亿，2.17 倍"
- ❌ "宇通有很深的护城河" → ✅ "对比同行，护城河体现在 XX 方面，差距 X%"
- ❌ "目标价 40 元" → ✅ "估值参考区间 25-40 元（不确定性 ±30%）"
- ❌ "83% 准确率" → ✅ "X 个样本无法给出统计显著的结论"
- ❌ "行业首个" → ✅ "公司宣称行业首个，待第三方验证"
- ❌ "未来会涨" → ✅ "如果满足 X 条件，估值可能修复到 Y 区间"
- ❌ "这家公司很好" → ✅ "这家公司相对同行的优势在 XX 方面，劣势在 YY 方面"

### 12.2 写作风格指南

**应该**：
- 用具体数字代替形容词
- 用对比代替孤立判断
- 用区间代替精确值
- 用反证测试代替单向论证
- 用可靠度标注代替权威断言

**不应该**：
- 用"很好""强大""领先""超越"等无对比基准的形容词
- 用"目标价 X 元"等暗示精确预测
- 用"行业首个"等营销话术
- 用"专家说""分析师认为"等权威背书

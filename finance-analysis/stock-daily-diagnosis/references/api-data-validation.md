# API 数据源可靠性验证（2026-06-30 实测案例）

## 案例：中科曙光 6/30 数据源差异

### 问题
东方财富 API 返回过时缓存数据，导致分析完全错误。

### 实际数据对比

| 数据源 | 6/30 收盘价 | 6/30 涨跌幅 | 6/30 成交额 |
|---|---|---|---|
| **腾讯 API（准确）** | **107.23 元** | **+10.00%（涨停）** | **153.52 亿** |
| 东方财富 API（缓存） | 99.46 元 | +1.67% | 150.71 亿 |
| **差异** | **7.77 元** | **8.33%** | **2.81 亿** |

### 根因分析
- 东方财富 `push2.eastmoney.com` 返回的是缓存/快照数据，不是实时数据
- 可能原因：API 限流、CDN 缓存、session 过期
- 腾讯 `qt.gtimg.cn` 始终返回实时数据

### 验证方法
```bash
# 腾讯实时行情
curl -s "http://qt.gtimg.cn/q=sh603019" | iconv -f gbk -t utf-8
# 字段 [3]=当前价, [4]=昨收, [30]=时间戳(YYYYMMDDHHmmss)

# 东方财富实时行情
curl -s -A "Mozilla/5.0" "https://push2.eastmoney.com/api/qt/stock/get?secid=1.603019&fields=f43,f60"
# f43=最新价(×100), f60=昨收(×100)
```

### 验证步骤
1. 同时拉取两个数据源的收盘价
2. 对比差异
3. 差异 < 0.01 元 → 数据可信
4. 差异 ≥ 0.01 元 → 以腾讯为准，标注"东方财富数据异常"

### 资金流向 API 限流
- 东方财富资金流向 API `push2.eastmoney.com/fflow` 会被限流（返回空数据）
- 限流时用外盘/内盘比例替代：
  - 外盘 > 内盘 → 买入主导
  - 外盘 < 内盘 → 卖出主导
- 腾讯 API 字段 [7]=外盘, [8]=内盘

### 腾讯 API 字段速查
```
[0] = 市场(1=SH, 0=SZ)
[1] = 股票名称
[2] = 股票代码
[3] = 当前价
[4] = 昨收
[5] = 今开
[6] = 成交量(手)
[7] = 外盘(手)
[8] = 内盘(手)
[30] = 时间戳(YYYYMMDDHHmmss)
[32] = 涨跌幅(%)
[35] = 最新价/成交量/成交额
[38] = 换手率(%)
[39] = PE(TTM)
```

# stock-weekly-forecast v1.0（A股个股未来一周趋势预测）

## 一、SKILL 的目的与边界

### 1.1 解决什么问题

对某只A股做**未来一周（5个交易日）**的趋势预测，给出**明确的概率判断和交易建议**。

**核心定位**：预测未来，不是诊断过去。

### 1.2 触发条件

**触发**：
- "XX 下周怎么走"
- "预测 XX 未来一周走势"
- "XX 接下来一周会涨还是跌"
- "帮我分析 XX 未来一周的机会"
- "做 XX 的一周趋势分析"

**不触发**（应使用 `stock-daily-diagnosis`）：
- "今日 XX 股价怎么样"
- "今天 XX 为什么涨/跌"
- "XX 现在什么情况"

**不触发**（应使用 `stock-deep-analysis`）：
- "分析 XX 半年走势"
- "XX 的估值是多少"

### 1.3 与 `stock-daily-diagnosis` 的关系

| 维度 | stock-weekly-forecast（本skill） | stock-daily-diagnosis |
|---|---|---|
| 核心问题 | "未来一周会怎样" | "现在什么状态" |
| 输出 | **概率预测 + 交易建议** | 状态诊断 + 等待条件 |
| 是否预测 | **必须预测** | 不预测 |
| 数据重点 | 融资融券 + 技术 + 事件 | 实时行情 + 资金流向 |
| 语气 | "大概率在 X-Y 区间运行" | "当前处于 X 状态" |

### 1.4 核心原则

1. **必须给出明确的概率预测**——不接受"中性偏多"这种模糊判断，必须说"55%概率先调整后反弹"
2. **必须给出具体价位区间**——如"94-105元区间运行"
3. **必须给出交易建议**——持有者/想买入者/空仓者分别怎么做
4. **必须给出止损止盈价位**——如"跌破92.16元止损"
5. **必须给出3种情景**——最可能/次可能/最小概率，各附概率和路径
6. **必须用API拉数据**——不能用搜索引擎估计
7. **必须说明数据缺口**——缺失什么数据、对判断有什么影响

## 二、A股一周窗口的核心认知

### 2.1 时间窗口特性

未来一周（5个交易日）是**噪声最大、信号最少**的窗口：
- 比1天长：纯消息套利失效
- 比1月短：估值锚定失效
- DCF/Fama-French/美林时钟**几乎不适用**

### 2.2 一周窗口下股价的真实驱动力排序

| 驱动力 | 权重 | 为什么 | 数据来源 |
|---|---|---|---|
| **资金面** | **30%** | A股散户占比60%+，资金动向≈短期价格 | 融资融券API |
| **事件催化** | 25% | 1周内事件能直接改变多空力量 | 公告/新闻 |
| **技术结构** | 20% | 趋势有惯性，支撑/压力位有效 | K线/均线 |
| **情绪/题材** | 15% | 涨停板/板块效应是A股独有 | 涨跌停统计 |
| 宏观背景 | 5% | 1周窗口影响有限 | 大盘指数 |
| 基本面 | 5% | 1周内业绩几乎不影响股价 | 财报 |

### 2.3 融资资金：A股一周窗口最强预测因子

**核心发现**：融资资金是A股短期走势的最强预测因子之一。

**为什么**：
- 融资是杠杆资金，买入方不愿亏钱→给股价提供下方支撑
- 融资余额变化反映"聪明钱"的方向
- 单日融资净买入异常放大（>正常水平3倍）是强烈信号

**判断规则**：
- 融资净买入连续3日为正 → 偏多
- 单日融资净买入异常放大（>3倍正常水平） → **强烈看多**
- 融资净偿还连续3日为正 → 偏空
- 融资余额从低位快速拉升 → 资金加速入场，短期有支撑

## 三、5阶段工作流

```
输入：股票代码 + 当前日期
   │
   ▼
[阶段 1] 宏观/市场环境扫描          5 分钟
   │  ├─ 大盘位置（上证/深证/创业板）
   │  ├─ 涨跌停数、上涨/下跌家数
   │  └─ 判断：顺风/逆风/中性
   ▼
[阶段 2] 个股资金面深度分析        10 分钟
   │  ├─ 融资余额变化（近10日）
   │  ├─ 融资净买入趋势
   │  ├─ 资金面拐点识别
   │  └─ 判断：偏多/偏空/中性
   ▼
[阶段 3] 个股事件扫描              10 分钟
   │  ├─ 近7日公告
   │  ├─ 评级/目标价变化
   │  ├─ 行业政策/事件
   │  └─ 判断：催化强/中/弱
   ▼
[阶段 4] 技术结构分析              10 分钟
   │  ├─ 均线系统（MA5/10/20）
   │  ├─ 关键价位（支撑/压力）
   │  ├─ 走势形态（涨停后调整/突破/盘整）
   │  ├─ 换手率/成交额
   │  └─ 判断：趋势向上/向下/震荡
   ▼
[阶段 5] 综合判断 + 概率预测       5 分钟
   │  ├─ 多空信号清单
   │  ├─ 3种情景 + 概率
   │  ├─ 具体价位区间
   │  ├─ 交易建议
   │  └─ 等待条件 + 止损止盈
   ▼
输出：未来一周趋势预测报告
```

## 四、阶段详细说明

### 阶段 1：宏观/市场环境扫描

**目的**：判断市场整体温度，确定"顺风"还是"逆风"。

**必拉数据**：

| 数据 | API 端点 |
|---|---|
| 三大指数 | `push2.eastmoney.com/api/qt/ulist.np/get?fltt=2&secids=1.000001,0.399001,0.399006&fields=f2,f3,f4,f12,f14` |
| 涨跌停统计 | `push2.eastmoney.com/api/qt/ulist.np/get?fltt=2&secids=1.000001&fields=f104,f105,f106,f107,f108,f109` |

**字段说明**：
- f2=最新价, f3=涨跌幅%, f4=涨跌额
- f104=上涨家数, f105=下跌家数, f106=涨停家数, f107=跌停家数

**判断规则**：
- 上证 > +1% 且涨停 > 40家 → **顺风**
- 上证 < -1% 且跌停 > 20家 → **逆风**
- 其他 → **中性**

### 阶段 2：个股资金面深度分析

**目的**：识别"聪明钱"方向。这是**最重要的阶段**。

**必拉数据**：

| 数据 | API 端点 |
|---|---|
| 融资融券（近10日） | `datacenter-web.eastmoney.com/api/data/v1/get?reportName=RPTA_WEB_RZRQ_GGMX&columns=DATE,RZYE,RZMRE,RZCHE,RZJME,RQYL,RQYE&filter=(SCODE=%22{代码}%22)&pageNumber=1&pageSize=10&sortTypes=-1&sortColumns=DATE&source=WEB&client=WEB` |

**关键字段**：
- `RZYE`: 融资余额（元）
- `RZMRE`: 融资买入额（元）
- `RZCHE`: 融资偿还额（元）
- `RZJME`: 融资净买入（= RZMRE - RZCHE，元）
- `RQYL`: 融券余量（股）
- `RQYE`: 融券余额（元）

**分析步骤**：
1. 计算近5日/3日融资净买入累计
2. 识别单日异常（>正常水平3倍）
3. 判断趋势（加速/减速/反转）
4. 给出资金面判断

**判断规则**：

| 情况 | 判断 |
|---|---|
| 近5日融资净买入 > 0 且趋势加速 | **强烈偏多** |
| 近5日融资净买入 > 0 但趋势减速 | **偏多** |
| 近5日融资净买入 < 0 且趋势加速 | **强烈偏空** |
| 单日融资净买入异常放大（>3倍） | **强烈看多信号** |

### 阶段 3：个股事件扫描（必做阶段，缺失则预测无效）

**目的**：识别过去1周已发生的催化（解释当前走势）+ 未来1周可能的催化。**这是整个工作流最容易被跳过的阶段，但一旦跳过，预测就退化为"历史诊断"——必须强制执行。**

**为何必做**：
- 不知道近期涨停/异动的原因，就无法判断趋势持续性
- 散户接盘+强催化 vs 散户接盘+无催化，结论可能完全相反
- 产业突破+政策支持+内部人信心三重共振下的"散户接盘"≠ 无催化下的"散户接盘"

**数据获取（按优先级）**：

| 优先级 | 数据 | 方法 | API/URL |
|---|---|---|---|
| P0 | 涨停原因（最近异动） | search_web + finance news | 财联社/东方财富新闻 |
| P0 | 近 7 日公司公告 | search_web `"{代码} 公告"` | 巨潮资讯/上交所/深交所 |
| P1 | 券商评级/目标价变化 | search_web `"{代码} 评级 研报"` | 同花顺/东方财富研报频道 |
| P1 | 行业政策/事件 | search_web `"行业政策 + {公司行业}"` | 政府官网/行业协会 |
| P2 | 业绩预告/订单 | search_web `"{代码} 业绩 订单 中标"` | 交易所/公司公告 |
| P2 | 龙虎榜（若上榜） | search_web `"{代码} 龙虎榜"` | 东方财富 |

**事件影响评估标准**（必须按此打分）：

| 影响等级 | 判断标准 | 对预测的修正 |
|---|---|---|
| **强正面** | 可能导致 > 3% 上涨（大额订单/业绩超预期/政策利好/技术突破/内部人信心） | 多头信号，权重 +2 |
| **正面** | 可能导致 1-3% 上涨（中标/评级上调/温和政策） | 多头信号，权重 +1 |
| **中性** | 影响有限（常规公告/小额中标/无关行业动态） | 不计入信号 |
| **负面** | 可能导致 1-3% 下跌（减持/评级下调/小利空） | 空头信号，权重 +1 |
| **强负面** | 可能导致 > 3% 下跌（业绩暴雷/重大利空/政策转向） | 空头信号，权重 +2 |

**常见重大催化类型**（按 A 股实证影响力排序）：

| 事件类型 | 权重 | 时间窗口 | 示例 |
|---|---|---|---|
| 业绩超预期/暴雷 | 强 | T+1 | 季报/年报 |
| 大额订单/中标 | 强 | T+1 | 10亿+订单 |
| 董监高增减持 | 强 | T+0~T+3 | 提前终止减持=强正面 |
| 股权激励/回购 | 中 | T+1~T+3 | 大额回购 |
| 解禁/减持 | 中 | T+0~T+3 | 大量解禁 |
| 行业政策（补贴/限制） | 强 | T+0~T+5 | 算力国家工程 |
| 评级上调/下调 | 弱 | T+1~T+3 | 大行评级 |
| 技术突破/产品发布 | 强 | T+0~T+5 | TOP500 第一 |

**事件新鲜度判断**（同样关键）：

| 新鲜度 | 判断标准 |
|---|---|
| **新鲜** | 今日公布，市场尚未反应 |
| **半新鲜** | 1-2 天前公布，市场部分反应 |
| **已消化** | 3+ 天前公布，市场已充分反应（不再作为当前催化） |
| **过期** | 1 周+ 前公布，不再是对当前走势的催化 |

**输出模板**（必须包含）：

```
| 日期 | 事件 | 影响等级 | 新鲜度 |
|------|------|----------|--------|
| 6/22 | 董高提前终止减持 | 强正面 | 新鲜 |
| 6/23 | 灵晟超算登顶 TOP500 | 强正面 | 半新鲜 |
| 6/24 | 万卡集群刷新 IO500 | 强正面 | 半新鲜 |
```

**如果无法获取**：明确说明数据缺口（"事件数据未获取"），但**必须基于公开信息推断至少 1-2 个可能催化**——不能完全跳过本阶段。

### 阶段 4：技术结构分析

**必拉数据**：

| 数据 | API 端点 |
|---|---|
| K线（近20日） | `push2his.eastmoney.com/api/qt/stock/kline/get?secid={secid}&fields1=f1,f2,f3,f4,f5,f6&fields2=f51,f52,f53,f54,f55,f56,f57,f58,f59,f60,f61&klt=101&fqt=0&beg={YYYYMMDD}&end={YYYYMMDD}` |
| 实时行情 | `push2.eastmoney.com/api/qt/stock/get?secid={secid}&fields=f43,f44,f45,f46,f47,f48,f60,f168,f169,f170` |

**secid规则**：
- 沪市（60开头）：`1.{代码}`
- 深市（00/30开头）：`0.{代码}`

**K线字段**（fields2=f51...f61）：
- f51=日期, f52=开盘, f53=收盘, f54=最高, f55=最低
- f56=成交量, f57=成交额, f58=振幅, f59=涨跌幅, f60=涨跌额, f61=换手率

**实时行情字段**：
- f43=今收, f44=最高, f45=最低, f46=今开
- f47=成交量(手), f48=成交额(元), f60=昨收
- f168=换手率, f169=量比, f170=PE(动)

**分析步骤**：
1. 计算MA5/MA10/MA20
2. 识别关键支撑/压力位
3. 判断走势形态
4. 分析换手率/成交额

### 阶段 5：综合判断 + 概率预测

**输出格式**：

```markdown
# {股票名}（{代码}）未来一周趋势预测

## 核心判断

**倾向：偏多/偏空/中性**

| 情景 | 概率 | 路径 | 目标价位 |
|---|---|---|---|
| 最可能情景 | XX% | 具体路径描述 | 支撑X，目标Y |
| 次可能情景 | XX% | 具体路径描述 | 目标Y |
| 最小概率情景 | XX% | 具体路径描述 | 止损Z |

**一句话预测**：未来一周大概率在X-Y区间运行，先A后B。

## 交易建议

**持有者**：...
**想买入者**：...
**空仓观望者**：...

## 关键观察点

| 观察点 | 偏多信号 | 偏空信号 |
|---|---|---|
| 开盘价 | 高开 > X元 | 低开 < Y元 |
| 成交额 | > X亿 | < Y亿 |
| 关键支撑 | 守住 | 跌破 |
| 关键压力 | 突破 | 受阻 |

## 等待条件
- 如果出现X，预测失效，转为偏空
- 如果出现Y，预测确认，加仓
```

## 五、关键规则

### 5.1 必须避免的错误

| 错误 | 正确做法 |
|---|---|
| 给出"中性偏多"这种模糊判断 | 给出"55%概率先调整后反弹" |
| 只诊断过去，不预测未来 | 必须给出未来一周的具体预测 |
| 不给交易建议 | 必须给持有者/买入者/空仓者分别建议 |
| 不给止损止盈 | 必须给出具体价位 |
| 不说明数据缺口 | 必须说明缺失什么数据、影响什么判断 |
| 用搜索引擎估计数据 | 必须用API拉取 |
| **跳过事件扫描（阶段 3）** | **预测等于历史诊断，无效** |
| **不知道近期异动原因就做预测** | **必须先查清楚最近一次涨停/异动的催化** |
| **把"散户接盘"无条件当作顶部信号** | **必须先看事件扫描结果——强催化下的散户接盘可能仍偏多** |

### 5.1b 输出前自我检查（Output Self-Check）

**生成报告前，必须逐项确认**：

- [ ] 5 阶段都执行了？（特别是阶段 3 事件扫描）
- [ ] 找到最近一次涨停/异动的催化原因？
- [ ] 给出了 3 种情景 + 概率？各情景概率之和 = 100%？
- [ ] 给出了具体价位区间（不是模糊的"可能涨可能跌"）？
- [ ] 给了持有者/想买入者/空仓者三类交易建议？
- [ ] 给出了明确的止损价位（具体数字，不是"跌破支撑"）？
- [ ] 给出了止盈价位或目标区间？
- [ ] 列出等待条件（什么数据出现时预测失效/确认）？
- [ ] 说明了数据缺口？

**任何一项缺失**：必须补齐才能输出。

### 5.2 概率判断的校准

- **无强信号**：向上/向下各约50%，给"震荡"判断
- **有单方面信号**：偏多/偏空约55-65%
- **多维度信号共振**：偏多/偏空约60-70%
- **趋势+资金+事件三重共振**：偏多/偏空约70%+

**注意**：即使最好的模型，一周窗口预测准确率也只有50-55%。不要过度自信。

### 5.3 一句话总结的写法规范

**格式**：[核心判断] + [关键价位] + [风险提示]

**示例**：
> 未来一周大概率在94-105元区间运行，先调整后反弹。融资资金6/26单日净买入+7亿（异常级别看多信号）提供下方支撑，100元整数关口是关键压力。风险：跌破92.16元则趋势破坏。

**禁止**：
**API数据源**：融资融券API（datacenter-web.eastmoney.com）、涨跌停统计API（push2.eastmoney.com）、K线API（push2his.eastmoney.com）

## 八、Pitfalls（高发错误）

### P1：把"未来一周预测"做成"过去一周诊断"

**症状**：用户问"下周怎么走"，agent 输出"过去几天发生了什么"。

**根因**：默认诊断过去比预测未来更"安全"，但用户问的是预测。

**修正**：每个段落都要回答"未来一周会怎样"。如果发现自己在写"过去 7 天 XX 涨了 XX%"，停下来——这是历史，不是预测。

### P2：跳过事件扫描（阶段 3）

**症状**：阶段 3 被静默跳过，直接进入技术分析和综合判断。

**根因**：阶段 3 数据源（公告/新闻）比 API 难调；agent 潜意识觉得"前 4 阶段够用了"。

**修正**：
- 强制使用 `search_web` 查询"XX 涨停原因"/"XX 公告"
- 至少找到最近一次异动的催化原因
- 即使数据不全也要基于公开信息推断至少 1-2 个催化

### P3：把"散户接盘"无条件判为顶部

**症状**：看到"小单+主力卖"就给出"看空"判断。

**根因**：散户接盘模式确实是经典顶部信号，但前提是**没有强催化**。

**修正**：
- 强催化下（产业突破+政策+业绩三重共振），散户接盘模式被压制
- 必须先看阶段 3 的事件扫描结果，再判断资金结构的含义
- 中科曙光 2026-06-30 案例：散户净买入 20 亿+小单，但因灵晟 TOP500 第一+80 亿加码 AI+董高终止减持三重催化，整体判断为偏多

### P4：把"融资单日异常放大"当作永久买入信号

**症状**：看到单日融资 +7 亿就给出"强烈看多"。

**根因**：融资异常放大确实是强烈信号，但需要看趋势持续性。

**修正**：
- 单日异常 + 后续 3 日净买入持续 → 持续性看多
- 单日异常 + 后续停止买入 → 已透支，杠杆资金观望
- 中科曙光 6/26 +7 亿是单日异常，但 6/29 转持平，**杠杆资金没有继续追**——信号强度从"强烈看多"降级为"偏多"

### P5：模糊的概率判断

**症状**：输出"中性偏多"/"震荡偏强"/"有上涨机会"。

**根因**：agent 担心明确的概率会被打脸，选择模糊表达。

**修正**：
- skill 明确禁止模糊判断
- 必须给出"XX% 概率路径 A，YY% 概率路径 B，ZZ% 概率路径 C"
- 三种情景概率之和 = 100%

### P6：止损位只说"跌破支撑"

**症状**：输出"跌破支撑止损"，没有具体数字。

**根因**：agent 知道有支撑，但不想承诺具体价位。

**修正**：
- 止损必须给具体数字：例"跌破 97.82 元止损"
- 优先选近 5 日内的关键价位（涨停价、前低、平台下沿）

## 九、与其他 skill 的关系

### 与 `stock-daily-diagnosis` 的差异

| 维度 | stock-weekly-forecast（本 skill） | stock-daily-diagnosis |
|---|---|---|
| 核心问题 | "未来一周会怎样" | "现在什么状态" |
| 输出 | **概率预测 + 交易建议** | 状态诊断 + 等待条件 |
| 是否预测 | **必须预测** | 不预测 |
| 数据重点 | 融资融券 + 技术 + **事件** | 实时行情 + 资金流向 |

**协作模式**：
1. 用户先用 `stock-daily-diagnosis` 诊断当前状态（回答"现在什么状态"）
2. 再用本 skill 预测未来一周（回答"下周怎么走"）
3. 两者交叉验证：当前状态 vs 未来预测是否一致

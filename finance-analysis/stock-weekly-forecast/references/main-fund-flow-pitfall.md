# 主力资金 API 字段定义 + 内部一致性校验实战指南

## ⚠️ 这是整个 A 股技能体系最容易踩的坑

东方财富 `fflow/daykline/get` 接口返回的资金流向数据，**直接用 f51 当主力净流入会得到错误的方向判断**。本参考文档详细说明字段定义、校验流程、以及真实案例。

## API 端点

```
GET https://push2.eastmoney.com/api/qt/stock/fflow/daykline/get
  ?lmt=5
  &klt=101
  &fields1=f1,f2,f3
  &fields2=f51,f52,f53,f54,f55,f56,f57,f58
  &secid=1.{代码}  # 沪市用 1.，深市用 0.
```

**返回示例**（中科曙光 2026-06-30）：
```json
{
  "klines": ["2026-06-30,1717810432.0,-546764128.0,-1171046160.0,-289555792.0,2007366224.0"]
}
```

返回 5 个字段，逗号分隔。

## 字段定义（关键）

| 字段 | 含义 | 单笔成交额阈值 |
|---|---|---|
| **f51** | 主力净流入（API 直接字段） | = 超大单 + 大单 |
| f52 | 超大单净流入 | ≥ 100 万元 / ≥ 50 万股 |
| f53 | 大单净流入 | 20-100 万元 / 10-50 万股 |
| f54 | 中单净流入 | 4-20 万元 / 2-10 万股 |
| f55 | 小单净流入 | < 4 万元 / < 2 万股 |

**单位**：所有字段单位是**元**，需要除以 1e8 转换为**亿元**。

## 东方财富"主力"标准定义

> **主力 = 超大单 + 大单 = f52 + f53**

这是东方财富、通达信、同花顺等软件通用的定义。**不要直接用 f51**，要按 f52+f53 重新计算。

## 【必做】3 项内部一致性校验

输出"主力净流入/流出"前必须做这 3 项检查：

### 校验 1：5 字段之和 ≈ 0

```
sum = f51 + f52 + f53 + f54 + f55
通过条件：|sum| < 0.5 亿
不通过：说明字段解码错误或数据异常，必须重读字段文档
```

**为什么**：理论上 5 字段（主力+超大单+大单+中单+小单）应该 = 0（流入 = 流出）。如果不 ≈ 0，要么是字段定义理解错，要么是数据异常。

### 校验 2：f51 vs (f52+f53) 方向一致

```
f51_sign = sign(f51)
calc_sign = sign(f52 + f53)
通过条件：f51_sign == calc_sign
不通过：以 (f52+f53) 为准（标准定义优先）
```

**为什么**：在某些情况下 f51 与按标准定义计算的"主力"方向相反。**永远以标准定义为准**。

### 校验 3：主力 vs 散户方向合理性

```
main = f52 + f53
retail = f55
合理性：main 和 retail 不应"同方向 + 同量级"（如主力+散户各买 20+ 亿）
```

**为什么**：在正常市场中，机构和散户应该方向相反或一个主导。如果同时大额同方向买/卖，要么数据有误，要么是极罕见的市场结构。

## 真实案例：中科曙光 2026-06-30

### 原始数据（单位：亿元）

| 字段 | 数值 | 含义 |
|---|---|---|
| f51 | +17.18 | API 直接给的"主力" |
| f52 | -5.47 | 超大单（机构最敏锐的资金） |
| f53 | -11.71 | 大单 |
| f54 | -2.90 | 中单 |
| f55 | +20.07 | 小单（散户） |

### 3 项校验结果

**校验 1：5 字段之和**
```
17.18 + (-5.47) + (-11.71) + (-2.90) + 20.07 = +17.17 亿
|17.17| > 0.5 → ❌ 异常
```

**校验 2：f51 vs f52+f53**
```
f51 = +17.18（正）
f52 + f53 = -5.47 + (-11.71) = -17.18（负）
方向相反 → ❌ 以 (f52+f53) 为准
```

**校验 3：主力 vs 散户**
```
主力（按标准定义）= -17.18 亿（机构在卖）
散户（小单）= +20.07 亿（散户在买）
方向相反，量级相当 → ✓ 合理（典型"散户接盘"模式）
```

### 正确解读 vs 错误解读

| 解读方式 | 结论 | 是否正确 |
|---|---|---|
| **错误**：直接用 f51 = +17.18 亿 | "主力买入 17 亿"（看多） | ❌ 完全相反 |
| **正确**：按 f52+f53 = -17.18 亿 | "主力卖出 17 亿，散户买入 20 亿（接盘）"（看空信号） | ✓ |

### 对最终判断的影响

| 维度 | 错误解读 | 正确解读 |
|---|---|---|
| 主力资金方向 | +17.18（买入） | -17.18（卖出） |
| 多空信号数 | 多 11 / 空 3 | 多 10 / 空 4 |
| 偏多概率 | 65% | 55-60% |
| 风险等级 | 中 | 中高 |

**错误解读会让概率虚高 5-10 个百分点**，错过必要的风险警示。

## Python 验证脚本模板

```python
def parse_main_fund_flow(raw_kline_str: str) -> dict:
    """解析东方财富 fflow/daykline/get 返回的资金流向数据，并做 3 项校验。"""
    parts = raw_kline_str.split(',')
    f51 = float(parts[0]) / 1e8  # 元 → 亿元
    f52 = float(parts[1]) / 1e8
    f53 = float(parts[2]) / 1e8
    f54 = float(parts[3]) / 1e8
    f55 = float(parts[4]) / 1e8

    # 标准定义的主力
    main_standard = f52 + f53

    # 校验 1：5 字段之和
    sum5 = f51 + f52 + f53 + f54 + f55
    check1_pass = abs(sum5) < 0.5

    # 校验 2：f51 vs f52+f53 方向一致
    f51_pos = f51 > 0
    main_pos = main_standard > 0
    check2_pass = f51_pos == main_pos

    # 校验 3：主力 vs 散户合理性
    same_direction = (f51 > 0 and f55 > 0) or (f51 < 0 and f55 < 0)
    same_magnitude = abs(f51) > 5 and abs(f55) > 5
    check3_pass = not (same_direction and same_magnitude)

    return {
        'f51_api_main': round(f51, 2),
        'main_standard': round(main_standard, 2),
        'super_large': round(f52, 2),
        'large': round(f53, 2),
        'medium': round(f54, 2),
        'small': round(f55, 2),
        'checks': {
            'sum5': {'value': round(sum5, 2), 'pass': check1_pass},
            'f51_consistency': {'f51_sign': '+' if f51_pos else '-', 'main_sign': '+' if main_pos else '-', 'pass': check2_pass},
            'main_retail_sanity': {'pass': check3_pass},
        },
        'final_main': round(main_standard, 2),  # 永远用标准定义
        'final_main_direction': '流入' if main_standard > 0 else '流出',
    }

# 中科曙光 6/30 案例
raw = "1717810432.0,-546764128.0,-1171046160.0,-289555792.0,2007366224.0"
result = parse_main_fund_flow(raw)
print(result)
# 输出:
# {
#   'f51_api_main': 17.18, 'main_standard': -17.18,
#   'super_large': -5.47, 'large': -11.71, 'medium': -2.90, 'small': 20.07,
#   'checks': {...all fail/sanity checks...},
#   'final_main': -17.18, 'final_main_direction': '流出'
# }
```

## 教训

1. **永远按标准定义重算**：不要相信 API 直接给的"主力"字段，先按 f52+f53 算一遍
2. **3 项校验是底线**：输出主力方向前必须做
3. **数学异常是信号**：5 字段之和异常通常意味着字段定义理解错误
4. **散户/主力合理性**：避免被"同买 37 亿"这种异常结构误导
5. **修正后风险评估更准确**：错误解读会让概率虚高 5-10 个百分点

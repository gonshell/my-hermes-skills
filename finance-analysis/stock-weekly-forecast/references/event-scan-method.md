# 事件扫描实战指南（v1.1 新增）

> 本文专门解决 skill 历史最高频的失败模式——**事件扫描阶段被跳过**。包含：必查清单、search_web 查询模板、不可用 API 列表、案例库。

## 为什么事件扫描是"必做"

中科曙光 2026-06-30 案例的核心教训：

- 仅看资金面：散户接盘（小单+20 亿，机构-17 亿）→ 判"顶部信号 → 偏空"
- 加入事件扫描：灵晟 TOP500 第一+6 万卡集群+80 亿加码 AI+董高终止减持 → **偏多 65%**

**结论**：没有事件扫描，预测会基于资金结构给出错误的方向。

## 必查清单（执行阶段 3 前先打印）

```
[ ] 最近 1 次涨停原因（最重要）
[ ] 最近 1 次异动原因
[ ] 近 7 日公司公告（业绩/订单/激励/减持）
[ ] 近期券商评级变化（看研报标题即可）
[ ] 行业重大政策/事件
[ ] 内部人动作（董监高增减持、回购）
```

**最少完成前 3 项才能进入阶段 4**。

## search_web 查询模板

### 查询 1：最近异动原因（必查）

```
"{代码} {股票名} {最近交易日} 涨停 原因"
"{代码} {股票名} 涨停 分析"
"{代码} 涨停揭秘"
```

**期望命中**：财联社/东方财富/同花顺的"涨停揭秘"类文章。

### 查询 2：近期公告

```
"{代码} 公告"
"{代码} 业绩预告 OR 中标 OR 订单 OR 激励"
"{股票名} 公告 巨潮"
```

### 查询 3：评级变化

```
"{代码} 评级 研报"
"{代码} 目标价 上调 OR 下调"
"{股票名} 中金 OR 中信 OR 华泰 评级"
```

### 查询 4：行业事件（按行业定制）

```
AI 算力类："算力 国家工程" + "{公司名}"
新能源类："{行业} 政策" + "{公司名}"
医药类："{公司名} 集采 OR 新药审批"
```

### 查询 5：内部人动作

```
"{代码} 减持 OR 增持 OR 回购"
"{公司名} 高管 增减持"
```

## 事件影响评估标准

| 等级 | 含义 | 调整预测概率 |
|---|---|---|
| 强正面 | >3% 上涨的潜在驱动 | 多头权重 +2 |
| 正面 | 1-3% 上涨 | 多头权重 +1 |
| 中性 | 影响有限 | 不计入 |
| 负面 | 1-3% 下跌 | 空头权重 +1 |
| 强负面 | >3% 下跌 | 空头权重 +2 |

## 事件新鲜度判断

| 新鲜度 | 判断标准 | 催化效力 |
|---|---|---|
| 新鲜 | 今日公布 | 强 |
| 半新鲜 | 1-2 天前 | 中 |
| 已消化 | 3+ 天前 | 弱 |
| 过期 | 1 周+ 前 | 不计入当前预测 |

## 实战案例库

### 案例 1：中科曙光 2026-06-30（强催化下散户接盘仍偏多）

**事件扫描结果**：
- 6/22 董高提前终止减持（强正面，新鲜）
- 6/23 灵晟超算登顶 TOP500 全球第一（强正面，半新鲜）
- 6/24 万卡集群刷新 IO500 全球纪录（强正面，半新鲜）
- 6/26 6 万卡 AI4S 集群在国家级节点投运（强正面，新鲜）
- 6/30 80 亿元加码 AI（强正面，新鲜）
- 中金维持"跑赢行业"评级（正面，新鲜）

**资金面**：
- 散户（小单）+20.07 亿（接盘）
- 机构（大单+超大单）-17.18 亿（卖出）

**结论**：散户接盘但因 6 重强催化（产业突破+政策支持+内部人信心），整体判偏多 65%。

**关键教训**：散户接盘 ≠ 一定看空。必须叠加事件扫描判断。

### 案例 2：典型"散户接盘+无催化 → 顶部信号"

如果同一时期没有上述事件，只有"散户接盘+机构出货"，应判**偏空**。

判断流程：
1. 资金面：散户接盘（看空信号）
2. 事件扫描：无催化（不构成反向对冲）
3. 综合：偏空

### 案例 3：单日融资异常+事件未明的处置

中科曙光 6/26 单日融资 +7 亿（异常）—— 4 天后（6/30）评估时：
- 6/27-29 融资转持平（异常信号未被强化）
- 6/30 散户接盘但有强催化
- 综合：单日融资异常强度从"强烈看多"降级为"偏多"（持续性不足）

**教训**：单日异常信号必须看后续是否持续，不能孤立使用。

## 不可用/受限的 API 列表

| 数据 | 端点 | 状态 | 替代方案 |
|---|---|---|---|
| 北向资金 | `push2.eastmoney.com/api/qt/kamtbs.wss` | 返回空 | 用融资数据部分弥补 |
| 主力资金多日趋势 | `push2.eastmoney.com/api/qt/stock/fflow/kline/get` | 只返回 1 天 | 仅用单日主力流向 |
| 龙虎榜明细 | 无公开 API | 需 search_web | 查"{代码} 龙虎榜 {日期}" |
| 公告原文 | 无公开 API | 需 search_web | 查"{代码} 公告 巨潮" |
| 研报原文 | 无公开 API | 需 search_web | 查"{代码} 研报 中金/中信" |

## 与 stock-daily-diagnosis 的事件影响表统一

两个 skill 共用同一套事件影响评估标准（5 等级）+ 新鲜度判断（4 等级）。这是刻意设计的——避免同一事件在两个 skill 里被不同打分。

如果未来调整标准，必须**同时**更新两个 skill 的相关章节。

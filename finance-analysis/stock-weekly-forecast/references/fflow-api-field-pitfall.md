# 东方财富 fflow API 字段踩坑案例（2026-06-30）

## 问题现象

调用 `push2.eastmoney.com/api/qt/stock/fflow/daykline/get?lmt=10&klt=101&fields1=f1,f2,f3&fields2=f51,f52,f53,f54,f55,f56,f57,f58&secid=1.603019`

返回原始数据（中科曙光 2026-06-30）：
```
1717810432.0,-546764128.0,-1171046160.0,-289555792.0,2007366224.0
```

5 个字段，对应 f51-f55：
- f51 = +17.18 亿（"主力净流入"）
- f52 = -5.47 亿（超大单净流入）
- f53 = -11.71 亿（大单净流入）
- f54 = -2.90 亿（中单净流入）
- f55 = +20.07 亿（小单净流入）

## 错误解读（agent 第一反应）

直接采用 f51 = +17.18 亿 → "主力净流入 +17.18 亿，机构在买入，散户（小单）也在买入 +20.07 亿，多空共振看多"

## 为什么是错的

**东方财富标准定义：主力 = 超大单 + 大单 = f52 + f53 = -17.18 亿（卖出）**

实际数据矛盾：
1. **5 字段之和 = +17.17 亿**（应该 ≈ 0，因为净额合计应该平衡）
2. **f51 = +17.18 亿 vs f52+f53 = -17.18 亿，符号完全相反**
3. **机构买 17 亿 + 散户买 20 亿 = 37 亿 单边买入**，这在物理上不合理

## 正确解读

主力（f52+f53）= -17.18 亿 → **机构净卖出 17 亿**
小单（f55）= +20.07 亿 → **散户净买入 20 亿**
结论：**典型"散户接盘"模式（顶部信号）**

## 教训

**不要直接信任 API 返回的第一个字段作为"主力"。** 东方财富不同接口的 f51 定义可能不同，必须用基本数学（5 字段求和）做内部一致性校验。

## 3 项必做校验

```
1. 5 字段之和 ≈ 0 校验：|f51+f52+f53+f54+f55| < 0.5 亿
2. f51 vs (f52+f53) 方向一致：符号相反时以 (f52+f53) 为准
3. 主力 vs 散户合理性：不应同方向 + 同量级（如主力+散户各买 20+ 亿）
```

如果任意一项不通过，**必须停下来重新读字段定义**，不要直接用 f51。

## 参考资料

- 搜索查询：东方财富超大单大单中单小单定义
- 标准定义：超大单 ≥ 50万股/100万元；大单 10-50万股/20-100万元
- "主力"= 超大单+大单 是行业惯例，不是某接口特殊定义

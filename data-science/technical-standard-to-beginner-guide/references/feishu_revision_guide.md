# 飞书文档修订操作规范

---

## 1. str_replace pattern 原则

飞书 docx XML 格式的 `str_replace` 指令要求：找到唯一匹配的文本片段，用新内容替换。

### 核心规则

1. **只替换纯中文短句**（不含 block ID）
2. **匹配文本必须唯一**：在同一文档中只出现一次
3. **精确到句子级别**，不要跨多行
4. **不要包含标点符号**（容易因全角/半角导致不匹配）

### 标准 pattern 写法

```python
{
    "str_replace": {
        "old_text": "要替换的文本",
        "new_text": "替换后的文本"
    }
}
```

### 常见错误

| 错误类型 | 原因 | 解决 |
|---------|------|------|
| `code: 1013, msg: pattern not found` | old_text 在文档中不存在 | 用 `docs +get-block-tree` 确认原文确切内容 |
| `code: 1013`（反复） | 文档已被修改，文本已变 | 先 re-read 文档获取最新内容再重试 |
| 替换后内容错位 | old_text 跨了多个 block | 拆分成多个更短的 pattern，每次只替换一个 block |

### 正确示例

```
# ✅ 正确：纯中文短句
old_text: "先导篇：为什么要制定这个标准"
new_text: "先导篇：为什么需要这个标准"

# ✅ 正确：术语定义
old_text: "固溶处理把铸件加热到高温让合金元素充分溶入铝基体"
new_text: "固溶处理：将铸件加热到高温，使合金元素充分溶入铝基体，然后快速冷却"
```

## 3. 常见错误码处理

### 1013 pattern not found

**原因：** `old_text` 与文档中实际内容不匹配（全角/半角、多了空格、换了说法）

**处理步骤：**
1. 调用 `lark-doc` 的 `docs +get-block-tree` 获取文档结构
2. 或者调用 `docs +read` 以 `full` 模式读取文档，找到确切文本
3. 复制原文的 exact 内容作为 `old_text`
4. 重试

### 1003 permission denied

**原因：** bot 对文档没有编辑权限

**处理步骤：**
1. 确认文档已授予 bot full_access 权限
2. 确认文档 owner 不是已被移除的账号
3. 通过 `lark-drive` 的 `docs +share` 重新授权

### 99991663 unknown block type

**原因：** 要插入的 block 类型文档不支持

**处理步骤：**
1. 飞书 docx 不支持某些 block 类型（如 callout、 toggle）
2. 将内容改为 paragraph 或 table 类型

### 230017 文档不存在或已被删除

**原因：** 文档 token 错误或文档被删除

**处理步骤：**
1. 用 `lark-doc` 的 `docs +search` 重新定位文档
2. 确认文档 token 未过期

## 4. 分块写入策略

当文档过大（>50KB）时，不要一次性 str_replace，分块处理：

```
第1步：写入先导篇 + 术语篇（第1~2层）
第2步：写入逐章解读（第3层）
第3步：写入表格解读 + 流程篇（第4层）
第4步：写入思考题
第5步：统一补充修订记录
```

每个块 str_replace 之前先 re-read 该块在文档中的确切内容，避免覆盖其他已写入的部分。

---

## 5. 写入后验证

每次写入完成，立即执行以下检查：

1. **读取验证：** 调用 `docs +read` 读取写入区域，确认内容已写入
2. **结构验证：** 确认标题层级正确（H1/H2/H3）
3. **权限验证：** 确认目标用户有 read 权限
4. **格式验证：** 表格、列表、代码块格式正确

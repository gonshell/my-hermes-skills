## 角色定义

| 角色 | 输入 | 输出 |

| 知识工程师 | HB5480-91 原文 + 5个 YAML | 逐条核对，PASS/FAIL |
| Python工程师 | diagnose.py + render_card.py | 硬编码扫描 + 架构违规 + 业务逻辑完整性 |

三个角色并行启动（delegate_task），各自独立，互不依赖。仲裁者收集三方结果后做最终决策。

## 仲裁规则

- 知识工程师：任何 FAIL → 冻结，等待修复
- Python工程师：硬编码任何数量 > 0 → 冻结
- QA工程师：UT 路由失败 > 0 → 冻结；DT 内容失败由仲裁者判断
- 三方全部 PASS → 通过

## 执行要点

**关键教训：测试用例格式必须适配最终用户场景。**

本技能运行在飞书频道，用户通过打字提问而非执行代码。
因此测试用例必须以**自然语言句式**呈现（如 `ZL105A T6固溶温度是多少`），
而非 Python 命令行格式。

验证时直接将该句式发给机器人，对比返回内容是否包含期望关键词。

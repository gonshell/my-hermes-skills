## 问题现象

用户问：`铸件X射线发现缺陷怎么判定？`

**期望**：返回缺陷判定流程引导（告诉用户需要提供缺陷类型+数量+厚度）

**实际**：返回"铸件是把金属液体浇进模具"的术语解释

---

## 根因分析

diagnose.py 第 626-648 行，else 分支的流程：

```
1. 检测术语匹配（在 terminology.yaml 中模糊搜索）
2. 若命中任一术语 → 返回 terminology_query
3. 若无任何匹配 → 返回 general（兜底提示）
```

问题：`铸件` 是 terminology.yaml 中的术语，匹配成功后直接返回术语解释，**跳过了用户真正想问的元问题**。

---

## 推荐修复（最小改动）

在 diagnose.py 第 626 行之前、术语匹配之前，插入元问题检测：

```python
# 在 # 术语查询：在 terminology.yaml 中匹配词条 之前插入
META_PATTERNS = [
    "怎么判定", "如何判断", "如何评定", "判定流程",
    "怎么检验", "如何检验", "标准是什么", "要求是什么"
]

for pattern in META_PATTERNS:
    if pattern in text:
        return {
            "type": "meta_guidance",
            "context": text,
            "content": (
                "要判定X射线缺陷，请提供以下信息：\n"
                "缺陷类型（如：偏析/裂纹/气孔/缩孔/冷隔）\n"
                "检出数量\n"
                "铸件厚度（mm）\n\n"
                "示例：'X射线发现1个偏析，厚度15mm'"
            ),
        }
```

对应的 render_card.py 需要新增 `_render_meta_guidance()` 函数。

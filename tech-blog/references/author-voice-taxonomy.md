# 作者声音污染分类法（Author Voice Taxonomy）

> 来源：2026-06 Neuralese 博客两轮清理实战。用户在初稿通过一致性检查 + 双轨审查 + 5 角色读者反馈后，指出"还是不能直接给读者看，内容有点乱入"，经过两轮系统清理总结出以下分类。

## 核心原则

文章里有**两条线**：
- **内容的声音** — 读者要的：事实、论证、判断
- **作者的声音** — 读者不要的：预告、标榜、辩护、抒情

发布前必须让前者盖过后者。零"作者声音"污染是硬标准。

---

## 5 型元叙事分类

### 1 型：预告自己要讲什么

**特征**：作者在正文里解释"接下来要写什么"，像目录摘要或行程单。

| 表现 | 例子 |
|------|------|
| 目录摘要 | "本文是 Neuralese 这门新语言的系统笔记：它从哪里来、它怎么工作…" |
| 章节预告 | "这一章梳理目前主流的三条技术路径" |
| 章节开场白 | "理论归理论。这一章看 2025–2026 年间的实践。" |

**处理**：删除整句，让正文直接开始讲内容。

### 2 型：标榜章节/证据地位

**特征**：作者替读者下结论，给章节或证据贴重要性标签或情绪标签。

| 表现 | 例子 |
|------|------|
| 自封地位 | "这一章是整篇文章的事实基石" |
| 最高级标签 | "最直接证据"、"最有力背书" |
| 情绪标签 | "这是最尴尬的反方观点" |
| 自评意义 | "LRT 是唯一…的方案——它的开源对社区跟进意义最大" |

**处理**：改为事实陈述。如"最直接证据" → 删掉最高级，只陈述事实。

### 3 型：解释文章结构

**特征**：作者解释自己的篇幅安排、排序逻辑、不对称设计。

| 表现 | 例子 |
|------|------|
| 解释篇幅 | "治理研究者的内容最长——因为这是最迫切的视角" |
| 命名正当性 | "给它一个新词是必要的——" |
| 结构自辩 | "这一章用三句话给三类读者。有意不平衡" |

**处理**：删除解释句。如果结构本身合理，读者自然能感受到；如果不合理，解释也没用。

### 4 型：稻草人再反驳

**特征**：否定一个没人提出的极端立场，或树立一个没来源的"常见误解"再推翻。

| 表现 | 例子 |
|------|------|
| 虚构误解 | "它的核心机制和'中间层 hidden state 替换 token'的常见误读不一样" |
| 否定式开头 | "它不是'外挂一个轻量网络'，而是…" |
| 极端立场反驳 | "不必把它当神迹膜拜" |

**处理**：删否定式开头，直接讲正确内容。

**注意例外**：技术对比中"不是 X，是 Y"有真实信息量时保留（如"Deep Think 不是多 agent 编排，而是单模型内部并行"——这里 X 是真实存在的误解）。

### 5 型：自我抒情收束

**特征**：作者自指文章结构 + 情感渲染。

| 表现 | 例子 |
|------|------|
| 自指结构 | "回到开头的比喻——" |
| 文学比喻过度 | "独立收敛到了同一片大陆。Neuralese 是那块大陆的轮廓。" |
| 情感渲染 | "这些团队的研究者们都值得被记住名字" |

**处理**：改为直述句。如"回到开头的比喻——模型有母语" → "**模型有母语**——这不是比喻。"

## 标题设计原则

### 禁止

- ❌ **冒号式标题**（`命名时刻：为什么必须有一个新词`）—— 用户原话"狗屁"，"看见哪个正式的文章标题有那么多冒号的？"
- ❌ **冒号统一**——不要用"阶段词：具体内容"格式来制造一致性
- ❌ **混搭风格**——比喻体（三岔口、盲点）和直述体（判断框架）交替
- ❌ **信息不自足**——标题需要上下文才知道讲什么（如"收束"、"一句话引入"）

### 正确做法

- ✅ **统一为名词短语**，4-7 字，零冒号、零括号
- ✅ **每个标题自身能传达章节核心信息**
- ✅ **所有标题单独读就是一条完整论证线**
- ✅ **统一是逻辑统一不是格式统一**

### 实例（Neuralese 博客，8 章标题）

```
0. 模型的母语          ← 抛出核心意象
1. Neuralese 的由来     ← 是什么
2. 显微镜下的概念空间   ← 证据
3. 三条技术路径         ← 怎么做
4. 思考的定价           ← 产品落地
5. 四个未解问题         ← 反方
6. 三类读者的判断       ← 怎么看
7. 不是终局             ← 收口
```

首尾呼应：章 0"模型的母语"抛出意象 → 章 7"不是终局"收束。

### 标题修正过程（反面教材）

```
第一版（混搭）           → 第二版（冒号统一）        → 第三版（名词短语）
─────────────────────    ─────────────────────────   ──────────────────
命名时刻：为什么…        命名：为什么必须…             Neuralese 的由来
技术路径三岔口            路径：三条互不兼容…           三条技术路径
四大盲点（反方/自我批判）  盲点：四个不能靠…            四个未解问题
收束                      收束：模型有母语              不是终局
```

第二版被用户否决（"狗屁"），因为冒号统一是机械统一——冒号前词不在同一语义层（"阶段"vs"领域"），且学术 PPT 腔重。

## 两轮清理模式

### 为什么需要两轮

第一轮（外科手术）通常只抓到**显性痕迹**：
- 章末小结（`> 小结：…`）
- 自检标记（`全文完。约5000字。证据闭环。`）
- 元叙事开头句（`这一章是整篇文章的事实基石`）

嵌入正文句子里面的元叙事（预告/标榜/稻草人/抒情）**第一轮看不到**——因为它们不是独立段落，是嵌在正常句子里的修饰语或插入语。

### 第二轮扫描方法

通读全文，逐段问：
> "这是给读者看的事实/论证/判断，还是作者的内心独白？"

用 5 型分类做检查清单：
1. 有没有预告"接下来讲什么"？→ 删
2. 有没有给章节/证据贴重要性标签？→ 改为事实陈述
3. 有没有解释自己的篇幅/排序安排？→ 删
4. 有没有否定一个没人提的立场？→ 删（保留有信息量的技术对比）
5. 有没有自指文章结构+抒情？→ 改为直述

### 预期效果

两轮合计净删约 2000 字（5000→3000 字级别），从"过程性文档"变成"面向读者的博客"。正文叙事骨架不动，删的全是"作者的声音"。

# Writing Style Extractor（写作风格提取器）

从标杆文章中提取可复用的结构化风格特征。这不是"讨论风格是什么"，而是从真实文本中**逐层拆解出可量化的写作 DNA**，输出一份可直接用于指导新文本生成的风格指南。

## 核心理念

**范例提取 ≠ 讨论生成。**

讨论生成会说："这位作者风格简洁有力，善用短句。"
范例提取会输出：

```
句长分布：均值 12 字，标准差 8 字，>30 字长句占比 < 5%
段落开头：72% 直接切入主题（"问题是……""上周……"），28% 设问开头
高频句式：主谓宾直述（67%），被字句（3%），把字句（12%）
```

区别在于：前者是形容词堆叠，后者是可操作的参数。

## When to Use

- 用户提供标杆/参考文章，要求"用这种风格写"
- 用户想分析自己或他人的写作风格特征
- 用户要建立品牌/个人写作风格指南
- 需要为新项目统一文案风格
- 配合 `humanizer` 技能使用：先提取风格，再用风格指南校准输出

**Don't use for:** 纯粹的文本润色（用 `humanizer`），内容改写（直接做），学术引用格式分析（不是本 skill 的范围）。

## 提取流程

### 第一步：获取源文本

源文本可能以三种方式提供：

1. **内联文本** — 用户直接粘贴在消息中
2. **文件** — 用户指定文件路径，使用 `read_file` 读取
3. **URL** — 用户给出链接，使用 `browser` 工具抓取正文

**最少样本量要求：** 至少 2000 字或 3 篇以上独立文章。不足时明确告知用户提取结论可能不稳定。

### 第二步：多维度扫描

对源文本执行以下 10 个维度的逐层扫描。每个维度必须给出**具体数值 + 范例**，而非泛泛描述：

#### 维度总览

| # | 维度 | 核心问题 | 输出类型 |
|---|------|---------|---------|
| 1 | 词汇层 | 用词偏好、词频、专业术语密度 | 词表 + 频率 |
| 2 | 句法层 | 句长分布、句式结构、修辞手法 | 统计 + 范例 |
| 3 | 段落层 | 段落长度、开头模式、收尾模式 | 模式分类 + 比例 |
| 4 | 篇章结构 | 整体框架、逻辑推进方式 | 结构图 |
| 5 | 修辞与语气 | 隐喻/类比/反问等、正式度、亲密度 | 修辞清单 + 语气标签 |
| 6 | 叙事视角 | 第一/二/三人称、叙述距离 | 比例 + 范例 |
| 7 | 节奏感 | 信息密度、张弛分布、长短交替 | 节奏图谱 |
| 8 | 文化锚点 | 引用来源、典故类型、受众预设 | 锚点清单 |
| 9 | 情感色彩 | 情绪基调、情绪波动模式 | 色彩图谱 |
| 10 | 标点习惯 | 标点符号使用频率与特殊用法 | 统计 + 特殊用法 |

详细的维度说明和提取方法见 `references/extraction-dimensions.md`。

### 第三步：交叉验证

在单维度扫描之后，执行交叉检查：

1. **一致性检查** — 各维度结论是否自洽（如声称"极简风格"但句长均值 35 字，需重新审视）
2. **异常值标记** — 找出偏离整体风格的片段，分析是有意为之还是噪声
3. **风格变体识别** — 同一作者可能在不同场景使用不同子风格（如开头庄重、中间活泼、结尾克制），标记切换点

### 第四步：输出结构化风格指南

按以下模板输出风格指南：

```markdown
# 写作风格指南：[作者/品牌名]

## 风格一句话定义
用一句话概括这个风格的核心特征（如："冷静克制的工程师叙事，信息密度极高，几乎零修辞"）。

## 1. 词汇特征
- 词表偏好：[高频词列表 top 20]
- 禁用/回避词：[作者明显回避的表达]
- 专业术语密度：[百分比]（附术语列表）
- 用词正式度：[1-5 分]（附范例）

## 2. 句法特征
- 句长分布：均值 [X] 字，中位数 [X] 字，标准差 [X]
- 句式偏好：[主谓宾/被字句/把字句/倒装等，附比例]
- 修辞手法清单：[使用频率 > 5% 的修辞，附范例]
- 句间连接方式：[并列/递进/转折/因果，附比例]

## 3. 段落特征
- 段落长度：均值 [X] 句，范围 [X-Y]
- 开头模式：[分类 + 比例 + 范例]
- 收尾模式：[分类 + 比例 + 范例]

## 4. 篇章结构
- 典型框架：[图示]
- 逻辑推进方式：[归纳/演绎/类比/时间线等]
- 过渡方式：[显性过渡词/隐性逻辑跳跃等]

## 5. 语气与修辞
- 正式度：[1-5 分]
- 亲密度：[1-5 分]
- 幽默度：[1-5 分]
- 标志性修辞：[列表 + 使用频率 + 范例]

## 6. 叙事视角
- 主要人称：[比例]
- 叙述距离：[近/中/远]
- 与读者关系：[平等对话/权威指导/朋友聊天等]

## 7. 节奏与密度
- 信息密度：[高/中/低]（每百字新概念数）
- 张弛分布：[紧凑段与舒缓段的比例]
- 长短交替规律：[描述]

## 8. 文化锚点
- 引用来源偏好：[书籍/电影/行业/科学等]
- 典故类型：[古典/流行/专业/个人经历]
- 受众预设：[专家/大众/特定圈层]

## 9. 情感色彩
- 基调：[描述]
- 情绪波动范围：[描述]
- 情感表达方式：[直接/间接/克制等]

## 10. 标点习惯
- 标点使用频率：[特殊标点的使用率]
- 破折号/省略号/感叹号用法：[描述]
- 中文标点 vs 西文标点偏好：[描述]

## 风格指纹
一段 200 字左右的文本，模仿该风格，作为快速校准参考。

## 反模式清单
列出该风格**不会**出现的特征（如"不会使用感叹号""不会出现排比句""不会使用网络流行语"），帮助排除偏差。
```

### 第五步：风格应用指导

在风格指南末尾附加一段「应用指导」，说明如何将此指南用于新文本创作：

1. **硬约束**（必须遵守）：3-5 条最核心的风格规则
2. **软约束**（倾向遵守）：5-8 条重要但可灵活处理的特征
3. **调参建议**：当需要适配不同场景时，哪些维度可以微调、怎么调

## 与 Humanizer 的协作

`writing-style-extractor` 和 `humanizer` 形成「风格提取 → 风格校准」的工作流：

1. 用 `writing-style-extractor` 从标杆文本提取风格指南
2. 用 `humanizer` 先去除新文本中的 AI 味
3. 将风格指南作为约束，对 humanize 后的文本做风格校准

## 多样本融合

当用户提供多篇标杆文章时：

1. **先分别提取**每篇的风格特征
2. **找出公共集**（所有样本共有的特征）— 这是最可靠的风格核心
3. **标记变体**（仅部分样本出现的特征）— 标注为"可选风格变体"
4. **输出融合指南**，区分核心风格和可选变体

## 常见陷阱

1. **样本量不足导致过拟合。** 仅凭一段 500 字的文本提取风格，结论极不稳定。必须提醒用户。
2. **混淆内容和风格。** "文章讨论了量子计算"是内容，"用类比解释技术概念"是风格。只提取后者。
3. **过度泛化。** "作者善用比喻"不够，要具体到"用日常物品比喻抽象概念（3 次/千字），不用神话/文学典故"。
4. **忽略文类差异。** 同一作者写技术博客和写朋友圈风格不同。必须标注提取的是什么文类下的风格。
5. **风格指纹不检验。** 输出风格指纹后，应对照各维度特征自检一遍，确保指纹确实体现了提取结论。
6. **形容词代替数据。** "简洁有力"不是提取结论，"句长均值 12 字，标准差 8"才是。每个论断必须有数据或范例支撑。

## 验证清单

- [ ] 源文本样本量 ≥ 2000 字或 ≥ 3 篇
- [ ] 10 个维度均已扫描并输出具体结论
- [ ] 每个维度的结论都有数据或范例支撑（非纯形容词描述）
- [ ] 交叉验证已完成，无自相矛盾
- [ ] 风格指南按标准模板格式输出
- [ ] 风格指纹段落存在且通过自检
- [ ] 反模式清单存在且非空
- [ ] 应用指导包含硬约束、软约束和调参建议
- [ ] 文类标注清楚（如"技术博客""散文""社交媒体"）

## One-Shot Recipes

### Recipe 1：从单篇文章提取风格

```
输入：用户粘贴一篇 3000 字文章
→ 执行 10 维度扫描
→ 交叉验证
→ 输出完整风格指南（含风格指纹 + 反模式清单）
→ 附加应用指导
```

### Recipe 2：从多篇文章提取统一风格

```
输入：用户给出 3 篇同一作者的文章
→ 分别对每篇做 10 维度扫描
→ 找出公共集和变体
→ 输出融合风格指南
→ 标注核心风格 vs 可选变体
```

### Recipe 3：提取风格 + 立即应用

```
输入：用户给标杆文章 + 要求用该风格写新内容
→ 先提取风格指南（精简版，重点关注硬约束）
→ 用风格指南作为约束生成新文本
→ 对照反模式清单检查生成结果
→ 输出新文本 + 风格匹配度自评
```

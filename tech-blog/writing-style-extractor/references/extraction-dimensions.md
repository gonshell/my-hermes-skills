# 写作风格提取维度详解

> 10个维度的详细提取方法，用于从参考文章中量化写作风格特征。

---

## 维度1：词汇（Lexicon）

**定义**：用词习惯，包括术语密度、正式度、中英混排模式。

**量化方法**：
- 术语密度 = 专业术语数 / 总词数 × 100%
- 正式度比率 = 书面语表达数 / (书面语+口语表达) × 100%
- 中英混排率 = 含英文的句子数 / 总句数 × 100%

**输出格式**：
```
词汇:
  术语密度: 12.3%（约每8词1术语）
  正式度: 高（85%书面语）
  中英混排: 频繁（42%句子含英文）
  高频术语: [FFmpeg, 编解码器, 容器格式, ...]
```

**中文特殊注意**：中文分词边界模糊，建议以"意群"为单位计算而非严格分词。成语、四字格算一个词汇单元。

**示例**：某技术博客术语密度15%，表明每6-7个词中有一个专业术语，属于"中等技术深度"。

---

## 维度2：句法（Syntax）

**定义**：句子结构和复杂度模式。

**量化方法**：
- 平均句长 = 总字符数 / 总句数（中文以句号/问号/叹号为分隔）
- 嵌套层级：统计每句中的逗号层级（0层=简单句，1层=1个从句，2+层=复杂从句）
- 主动/被动比 = 主动句数 / 被动句数

**输出格式**：
```
句法:
  平均句长: 35.2字（偏长）
  简单句占比: 40%
  复合句占比: 45%
  嵌套句占比: 15%
  主动/被动比: 4:1
```

**中文特殊注意**：中文被动句常省略"被"字（如"问题解决了"），需语义判断。

---

## 维度3：段落（Paragraph）

**定义**：段落长度和结构模式。

**量化方法**：
- 段均长度 = 总字符数 / 总段落数
- 结构模式统计：统计"总-分"、"并列"、"递进"、"转折"等模式的出现频率
- 过渡词密度 = 过渡词数 / 总段落数

**输出格式**：
```
段落:
  段均长度: 120字（中等）
  段落结构分布: 总分45% / 并列30% / 递进20% / 转折5%
  过渡词: 使用频繁（首先/其次/最后/因此/然而）
```

**中文特殊注意**：中文段落间常省略显式过渡词，依靠语义自然衔接。

---

## 维度4：篇章（Discourse）

**定义**：整体叙事结构和章节组织方式。

**量化方法**：
- 叙事结构类型识别：线性/倒叙/插叙/并列式/螺旋式
- 章节长度标准差：衡量各章节篇幅是否均匀
- 开头模式：场景描写/问题引入/数据冲击/名言引用
- 结尾模式：总结回扣/展望未来/行动号召/开放问题

**输出格式**：
```
篇章:
  叙事结构: 线性递进（问题→原理→实践→展望）
  章节长度: 均匀（标准差±15%）
  开头模式: 问题引入（"你是否想过..."）
  结尾模式: 展望未来
```

**中文特殊注意**：中文技术博客常用"引子→正文→总结"三段式，比英文更结构化。

---

## 维度5：修辞（Rhetoric）

**定义**：修辞手法和说明策略的使用模式。

**量化方法**：
- 比喻/类比频率 = 比喻数 / 总段落数
- 排比/对偶频率 = 排比结构数 / 总段落数
- 数据驱动比 = 含具体数据的句子 / 总论点数
- 感性描述比 = 感性描述段落 / 总段落数

**输出格式**：
```
修辞:
  比喻频率: 中等（约每3段1个比喻）
  典型比喻: "编码器就像翻译官"
  数据驱动: 高（80%论点有数据支撑）
  感性描述: 低（10%）
```

**中文特殊注意**：中文技术博客常用类比但较少用隐喻；四字格（如"举一反三"）算隐性修辞。

---

## 维度6：视角（Perspective）

**定义**：叙事人称和视角选择。

**量化方法**：
- 人称分布：统计第一人称（我/我们）、第二人称（你/你们）、第三人称（他/它）的出现频率
- 视角一致性：检查全文是否固定使用一种主要人称

**输出格式**：
```
视角:
  主要人称: 第一人称+第二人称混合
  分布: 第一人称30% / 第二人称50% / 第三人称20%
  一致性: 高（全文人称统一）
```

**中文特殊注意**：中文技术博客中"你"常作为泛指代词使用，不一定指向读者。

---

## 维度7：节奏（Rhythm）

**定义**：信息密度的变化模式和阅读节奏感。

**量化方法**：
- 长短句交替率 = 长短句交替次数 / 总句数（长句>40字，短句<15字）
- 信息密度变化：按段落统计专业术语/新概念的出现密度
- 高潮标记：识别信息密度峰值段落

**输出格式**：
```
节奏:
  长短句交替: 频繁（交替率35%）
  信息密度: 波浪型（每3段一个密度峰）
  节奏模式: 快-慢-快（技术段-过渡段-技术段）
```

**中文特殊注意**：中文四字格天然具有节奏感，出现频率影响整体节奏感知。

---

## 维度8：文化（Culture）

**定义**：文化背景假设、引用来源分布。

**量化方法**：
- 引用来源分布：中文源/英文源/日文源的比例
- 文化背景假设：统计隐含的文化背景知识（如假设读者了解某产品/事件）
- 行业黑话密度 = 行业特有术语 / 总术语数

**输出格式**：
```
文化:
  引用来源: 中文40% / 英文55% / 日文5%
  文化假设: 假设读者了解互联网行业基本概念
  行业黑话: 中等密度（如"DAU""灰度""AB测"）
```

**中文特殊注意**：中文技术博客常混用中英文术语，需区分"必要英文"（无中文对应）和"习惯英文"（有中文但惯用英文）。

---

## 维度9：情感（Sentiment）

**定义**：情感极性和强度模式。

**量化方法**：
- 情感极性：正向/中性/负向段落的占比
- 情感强度：1-5级（1=完全客观，5=强烈情感）
- 幽默/严肃比 = 含幽默元素的段落 / 总段落数

**输出格式**：
```
情感:
  极性分布: 正向40% / 中性50% / 负向10%
  平均强度: 2.3/5（偏理性客观）
  幽默元素: 罕见（约每10段1处）
```

**中文特殊注意**：中文含蓄表达多，讽刺/幽默常通过反问、夸张等间接方式呈现。

---

## 维度10：标点（Punctuation）

**定义**：标点使用习惯和特殊标点模式。

**量化方法**：
- 标点频率分布：逗号/句号/顿号/分号/冒号/破折号/省略号/感叹号的使用频率
- 特殊标点模式：省略号、破折号、感叹号的使用场景
- 中英文标点混用检查

**输出格式**：
```
标点:
  主要标点: 逗号55% / 句号30% / 顿号8% / 其他7%
  破折号: 偶尔用于插入解释
  省略号: 几乎不用
  感叹号: 仅出现在章节标题
  中英标点: 统一使用中文标点
```

**中文特殊注意**：中文独有的顿号（、）和书名号（《》）是风格特征的重要指标。

---

*维度详解完*

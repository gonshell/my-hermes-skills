# 一致性检查详细规则

本文档定义 `chapter-consistency-checker` 各维度的详细检查规则和判定标准。

---

## 1. 术语一致性规则

### 1.1 同义异形检测

| 规则ID | 规则描述 | 判定标准 |
|--------|----------|----------|
| T-001 | 同一概念使用不同名称 | 跨章节出现2个以上不同表述指向同一概念 → 🟡 |
| T-002 | 首次出现未定义 | 专业术语首次出现无解释/定义 → 🟡 |
| T-003 | 缩写使用不统一 | 同一术语有时用全称有时用缩写 → 🟡 |
| T-004 | 英中文混用不一致 | 同一术语有时英文有时中文无规律 → 🟢 |
| T-005 | 术语拼写变化 | 同一英文术语拼写不一致（color/colour）→ 🔴 |

### 1.2 术语提取方法

1. 提取所有**专业名词**（含中英文）
2. 建立**同义词组**（人工确认或上下文推断为同一概念）
3. 检查每组的**使用一致性**

### 1.3 常见不一致模式

- 中英混用：「前端」vs 「Frontend」vs 「front-end」
- 缩写不统一：「API」vs 「Application Programming Interface」
- 译名差异：「设计模式」vs 「Design Patterns」
- 大小写不一致：「JavaScript」vs 「Javascript」vs 「javascript」

## 2. 语气一致性规则

### 2.1 语气分类光谱

```
学术 ←—→ 通俗 ←—→ 对话 ←—→ 叙事
  ↑         ↑         ↑         ↑
严谨      专业      轻松      文学
客观      清晰      亲切      感性
被动多    主动多    口语化    描写多
```

### 2.2 语气判定指标

| 指标 | 学术语气 | 通俗语气 | 对话语气 | 叙事语气 |
|------|----------|----------|----------|----------|
| 代词使用 | 避免第一人称 | 适度「我们」 | 频繁「你/我」 | 角色名 |
| 感叹号 | 几乎不用 | 偶尔使用 | 经常使用 | 适度使用 |
| 疑问句 | 很少 | 适度引导 | 频繁互动 | 对话中多 |
| 缩略语 | 不用 | 适度 | 常用 | 视风格 |
| 证据引用 | 频繁 | 适度 | 少 | 视题材 |
| 比喻类比 | 谨慎使用 | 常用 | 日常比喻 | 文学比喻 |

### 2.3 语气漂移判定

| 规则ID | 规则描述 | 判定标准 |
|--------|----------|----------|
| O-001 | 章节间语气跳跃 | 相邻章节语气分类跨度≥2 → 🔴 |
| O-002 | 段落内语气突变 | 同段内语气指标矛盾≥3项 → 🟡 |
| O-003 | 称呼读者方式不一致 | 「你/您/读者」混用 → 🟡 |
| O-004 | 情感强度不均 | 部分章节情感极端而其他平淡 → 🟢 |

## 3. 细节层次一致性规则

### 3.1 细节层次分级

| 层级 | 描述 | 特征 |
|------|------|------|
| L1-概述 | 宏观介绍 | 无代码/公式，概念为主 |
| L2-解释 | 中等深度 | 有示例，无完整代码 |
| L3-详解 | 深入展开 | 完整代码/公式/图表 |
| L4-实战 | 极度详细 | 完整项目/步骤/输出 |

### 3.2 均衡性检查

| 规则ID | 规则描述 | 判定标准 |
|--------|----------|----------|
| D-001 | 相似主题深度差异 | 同类主题章节深度差≥2级 → 🟡 |
| D-002 | 篇幅失衡 | 某章节篇幅<平均篇幅的30% → 🟡 |
| D-003 | 示例密度不均 | 某章节无示例而同类章节≥3个 → 🟡 |
| D-004 | 代码/图文比例失衡 | 代码密集章节与纯文字章节交替 → 🟢 |

## 4. 人称一致性规则

### 4.1 人称分类

| 人称 | 中文标志词 | 适用场景 |
|------|-----------|----------|
| 第一人称单数 | 我 | 个人经验、教程 |
| 第一人称复数 | 我们 | 团队协作、引导式教学 |
| 第二人称 | 你/您 | 教程、指南 |
| 第三人称 | 他/她/它/作者 | 学术、客观叙述 |
| 无主语/被动 | （省略） | 学术、技术文档 |

### 4.2 人称检查规则

| 规则ID | 规则描述 | 判定标准 |
|--------|----------|----------|
| P-001 | 全文人称不统一 | 主人称在章节间切换≥2次 → 🔴 |
| P-002 | 段落内人称跳跃 | 同段内出现2种以上人称 → 🟡 |
| P-003 | 「我们」使用不一致 | 有时代表作者有时代表读者 → 🟡 |
| P-004 | 被动/主动语态比例突变 | 相邻章节语态比例差>40% → 🟢 |

## 5. 引用一致性规则

### 5.1 引用格式标准

#### 行内引用格式
- 学术风格：`[1]`、`[Author, Year]`、`(Author, Year)`
- 技术文档：`[^footnote]`、超链接 `[文字](url)`
- 混合格式：`[文字][ref]` + 尾注

#### 参考文献格式
- **APA**：Author, A. A. (Year). Title. *Journal*, Volume(Issue), Pages.
- **IEEE**：[1] A. Author, "Title," *Journal*, vol. X, no. Y, pp. Z, Year.
- **GB/T 7714**：作者. 题名[J]. 刊名, 年, 卷(期): 页码.

### 5.2 引用检查规则

| 规则ID | 规则描述 | 判定标准 |
|--------|----------|----------|
| C-001 | 引用格式混用 | 同一文档出现≥2种引用格式 → 🔴 |
| C-002 | 参考文献格式不一致 | 列表中格式差异≥3处 → 🔴 |
| C-003 | 同一引用重复编号 | 同一文献有2个以上编号 → 🟡 |
| C-004 | 编号不连续 | 引用编号跳号 → 🟡 |
| C-005 | 图片/表格编号格式不一致 | 有的用「图1」有的用「Figure 1」→ 🟡 |
| C-006 | 链接引用vs文字引用混用 | 无规律的引用方式切换 → 🟢 |
| C-007 | 悬空引用 | 文中引用了但参考文献列表中无对应 → 🔴 |
| C-008 | 未引用的参考文献 | 列表中有但文中未引用 → 🟡 |

## 严重性判定总览

| 级别 | 图标 | 含义 | 处理建议 |
|------|------|------|----------|
| 🔴 严重 |  | 直接影响理解或专业性 | 必须修改 |
| 🟡 中等 |  | 影响阅读体验或规范性 | 建议修改 |
| 🟢 轻微 |  | 风格细微差异 | 可选修改 |

## 特殊情况处理

### 有意的不一致

当作者有意在不同章节使用不同风格时（如教程章节用对话语气，参考手册用学术语气）：
1. 标注为「疑似有意不一致」
2. 不计入问题数
3. 在报告中单列「有意设计」章节

### 翻译作品

翻译作品可能出现原文术语保留的情况：
1. 保留原文术语不算不一致
2. 但需检查是否在所有章节都保留
3. 标注为「原文保留」并确认一致性

### 多作者协作

多作者作品出现风格差异是正常的：
1. 降低语气不一致的严重性（🟡→🟢）
2. 重点关注术语和引用一致性
3. 建议建立统一的风格指南

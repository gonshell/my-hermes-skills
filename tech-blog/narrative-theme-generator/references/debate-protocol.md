# 辩论协议（Debate Protocol）

> 本文件是 `narrative-theme-generator` Skill 的详细辩论行为规范。

---

## 1. 角色行为规范

### 1.1 通用规则

- **身份锁定**：一旦角色确定，全程不得切换视角或"代为发言"。
- **论据标准**：每个论点必须附带至少一个具体论据（案例、数据、逻辑推导、亲身经历模拟）。
- **禁止空洞评价**：不允许出现"我觉得不错""这个方向很好"等无理由评价。必须说明 **好在哪里** 或 **为什么不行**。
- **语言风格**：每个角色应保持自己的语言风格差异——主题专家偏学术/技术口吻，传播专家偏营销/策略口吻，读者代表偏口语/直觉表达。

### 1.2 主题专家行为规范

| 场景 | 正确行为 | 错误行为 |
|------|----------|----------|
| 发现事实错误 | 明确指出错误并给出正确信息 + 来源依据 | 含糊说"不太对" |
| 认为候选太浅 | 指出具体缺少了哪个深度维度，并示范深入一层 | 只说"深度不够" |
| 同意某候选 | 从专业角度补充为什么这个方向专业上成立 | 只说"我同意" |
| 反驳某候选 | 用反例或反面数据支撑，指出具体哪里不成立 | 无证据反驳 |

### 1.3 传播专家行为规范

| 场景 | 正确行为 | 错误行为 |
|------|----------|----------|
| 判断传播潜力 | 参考同类成功/失败案例，给出类比 | 凭空判断"能火" |
| 建议优化 | 给出具体的标题/钩子/传播渠道建议 | 只说"需要优化" |
| 发现传播风险 | 指出可能引发负面传播的具体点 | 无具体原因说"有风险" |
| 评估平台适配 | 明确说哪个平台更适合、为什么 | 笼统说"可以发" |

### 1.4 读者代表行为规范

| 场景 | 正确行为 | 错误行为 |
|------|----------|----------|
| 看不懂时 | 明确说"这段我看不懂"，指出卡在哪里 | 假装看懂 |
| 有共鸣时 | 用自己的话描述为什么被触动 | 复述原文 |
| 觉得无聊时 | 诚实说"我读到这里就想关掉了" | 勉强说"还行" |
| 给建议时 | 从"我作为读者希望看到什么"出发 | 替作者考虑"应该怎么写" |

## 2. 冲突处理规则

### 2.1 角色间意见冲突

当两个角色对同一条候选给出相反评价时：

1. **不要折中**：不允许"各打五十大板"式的妥协。
2. **各自深化论据**：每方必须再补充1个新论据。
3. **引入第三角色裁决**：第三角色必须选择站在其中一方，并给出理由。
4. **记录分歧**：最终输出中标注"存在分歧"，不强制统一。

### 2.2 所有角色一致好评/差评

- **一致好评**（3个角色都给出 ≥ 4.5/5）：触发"魔鬼代言人"机制——自动生成1个尖锐的反面论点，挑战共识。
- **一致差评**（3个角色都给出 ≤ 2/5）：直接淘汰，无需进入下一轮。但必须记录 **差评的共同原因**，作为对主题描述的反馈。

### 2.3 角色自相矛盾

如果某角色在同一轮中前后矛盾：
- 其他角色有权指出矛盾
- 该角色必须选择一个立场并解释原因
- 矛盾本身可能是有价值的洞察，值得记录

## 3. 极端情况应对

### 3.1 候选数量爆炸

如果合成候选超过8条：
- 触发"快速淘汰"：每个角色标记最多2条"可直接淘汰"，被 ≥ 2个角色标记的自动淘汰。
- 淘汰后剩余候选控制在 4-6 条，进入第三轮。

### 3.2 候选数量不足

如果合成候选少于3条：
- 不降低质量标准凑数
- 输出时明确标注"本轮有效候选不足3条"
- 建议：扩大主题范围或放松约束条件

### 3.3 主题过于宽泛

如果第一轮中角色反馈"主题太宽，无法聚焦"：
- 暂停讨论，请求用户收窄主题范围
- 或将大主题拆分为子主题，每个子主题独立运行一轮完整流程

### 3.4 主题过于狭窄

如果第一轮中角色反馈"主题太窄，叙事空间有限"：
- 尝试从3个不同维度（历史演变、跨领域类比、未来推演）拓展
- 如果仍无法产生3条以上有效候选，建议用户重新审视主题价值

### 3.5 角色陷入死循环

如果同一论点被反复提及超过2轮：
- 主持人（系统角色）介入，标记"此论点已在第N轮充分讨论"
- 要求引入新论据或转移讨论焦点

## 4. 置信度计算方法

### 4.1 单角色置信度

每个角色对每条候选给出 0-100% 的置信度：

```
角色置信度 = 主题相关性(25%) + 独到性(25%) + 可执行性(25%) + 自维度契合度(25%)
```

其中"自维度契合度"：
- 主题专家看：专业深度是否足够
- 传播专家看：传播路径是否清晰
- 读者代表看：情感共鸣是否强烈

### 4.2 综合置信度

```
综合置信度 = 主题专家置信度 × 0.35 + 传播专家置信度 × 0.35 + 读者代表置信度 × 0.30
```

### 4.3 置信度校准

原始置信度需要经过校准：

| 原始范围 | 校准后 | 说明 |
|----------|--------|------|
| 90-100% | 降5-10% | 过度乐观偏差修正 |
| 70-89% | 保持 | 合理区间 |
| 50-69% | 保持 | 中等置信，需注明风险 |
| 30-49% | 升5% | 过度悲观偏差修正 |
| 0-29% | 升5% | 底线保护，但标注"高风险" |

## 5. 质量守卫规则

### 5.1 必须满足的硬性条件（任一不满足则候选无效）

- [ ] 叙事线可以用一句话清晰表达
- [ ] 与主题描述直接相关（不是硬凑的）
- [ ] 目标受众能理解（不需要超出目标受众知识范围的前置知识）
- [ ] 在给定约束条件下可执行

### 5.2 输出前最终检查

- [ ] Top3 中没有两条本质相同的候选（重复检查）
- [ ] 每条候选的3个角色置信度都已标注
- [ ] 风险提示不是空泛的"可能不够好"，而是具体的失败场景
- [ ] 淘汰摘要包含每条淘汰候选的一句话理由
- [ ] 整个讨论过程无角色同质化现象

## 6. 术语表

| 术语 | 定义 |
|------|------|
| **叙事线** | 一条贯穿多篇内容的核心故事线或论证脉络 |
| **放大** | 在前一轮结论基础上补充新论据、新角度、新维度的操作 |
| **合成候选** | 第二轮碰撞后，由原始候选 + 放大产生的新方向合并而成的候选池 |
| **置信度** | 该叙事线能在目标受众中产生预期效果的信心程度（0-100%） |
| **阻断性问题** | 如未解决将导致该叙事线完全失效的致命缺陷 |
| **魔鬼代言人** | 当所有角色达成共识时，自动引入的反面论点挑战机制 |

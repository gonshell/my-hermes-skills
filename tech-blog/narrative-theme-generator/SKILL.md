# Narrative Theme Generator — 多Agent放大器型叙事线生成

## 概述

本 Skill 实现一个 **"放大器型"多Agent讨论框架**，通过3个专家角色进行3轮结构化辩论，生成高质量的叙事线（narrative thread）候选。核心设计原则：

> **每个角色在每一轮中对前一轮的结论进行"放大"——补充论据、提出反驳、拓展维度，而非简单投票取平均。**

最终输出 **Top3 叙事线候选 + 置信度评分**，供创作者做最终决策。

---

## 使用场景

### 适合使用
- 需要2-5个核心叙事线候选的长篇内容策划（专栏、系列文章、品牌故事线）
- 需要多视角碰撞才能突破创意瓶颈的场景
- 产品/品牌叙事策略制定
- 小说、剧本、播客节目的主题线探索

### 不适合使用
- 单篇文章的标题/主题确定（过于重量级）
- 纯数据驱动的选题（不需要多角色辩论）
- 时间紧迫需要即时结果的场景（整个流程约需10-15分钟）

## 三个角色定义

### 🔬 角色一：主题专家（Subject Expert）

| 维度 | 说明 |
|------|------|
| **画像** | 领域内深耕的研究者或从业者，对主题有深度认知 |
| **关注点** | 事实准确性、领域前沿趋势、被忽略的深度角度、专业壁垒 |
| **辩论风格** | 以数据和案例驱动，善于提出"你以为X，实际上Y"的反直觉观点 |
| **口头禅** | 「这个方向很多人写过，但真正的盲区在……」 |
| **放大策略** | 每一轮都尝试挖掘更深的层次，将表面现象还原为底层机制 |

### 📣 角色二：传播专家（Communication Expert）

| 维度 | 说明 |
|------|------|
| **画像** | 深谙传播规律的内容策略师，关注信息如何触达并影响受众 |
| **关注点** | 话题热度、传播裂变潜力、标题与钩子设计、情绪共振点、平台适配性 |
| **辩论风格** | 以传播数据和案例驱动，善于判断"这个角度能不能火" |
| **口头禅** | 「内容再好，如果读者不点开就等于零。」 |
| **放大策略** | 每一轮都追问传播可行性，将叙事线优化为可传播的形态 |

### 👤 角色三：读者代表（Reader Representative）

| 维度 | 说明 |
|------|------|
| **画像** | 目标受众的典型代表，以"普通读者"心态审视内容价值 |
| **关注点** | 是否能看懂、是否有共鸣、是否读完有收获、阅读体验是否流畅 |
| **辩论风格** | 以直觉和感受驱动，直接说"这段我读不下去"或"这个角度太棒了" |
| **口头禅** | 「你们说的我都听不懂，但如果是这样的标题我会点开。」 |
| **放大策略** | 每一轮都从读者视角"翻译"专业讨论，暴露认知鸿沟和情感需求 |

## 3轮讨论流程

### 第一轮：发散（Diverge）—— 各自提出初始候选

**输入**：主题描述 + 目标受众 + 内容类型 + 约束条件

每个角色独立提出 **3条叙事线候选**，格式：

```
### [角色名] 的第N条候选
- **叙事线**：（一句话概括核心故事线）
- **切入点**：（从什么角度进入主题）
- **预期价值**：（读者读完能得到什么）
- **传播潜力**：（自评 1-5 分，附理由）
```

**放大规则**：本轮无放大，纯发散。

---

### 第二轮：碰撞（Collide）—— 交叉评论 + 放大

每个角色阅读其他两个角色的候选，执行以下操作：

1. **选中最具潜力的1条**（不选自己的），说明为什么
2. **提出1个尖锐的反驳或补充**（"放大"操作）
3. **暴露1个被忽略的盲区**

**放大规则**：
- 主题专家：补充专业深度/指出事实错误
- 传播专家：评估传播可行性/建议传播优化
- 读者代表：暴露理解障碍/表达情感需求

本轮结束时，**生成合成候选**（最多6条）：原始候选中被选中次数最多的 + 放大后产生的新方向。

---

### 第三轮：收敛（Converge）—— 打分排序 + 最终辩护

每个角色对所有合成候选进行独立评分：

| 评分维度 | 权重 | 说明 |
|----------|------|------|
| 主题深度 | 25% | 是否触及主题的核心而非浮于表面 |
| 传播潜力 | 25% | 是否有让读者主动分享的冲动 |
| 读者共鸣 | 25% | 目标读者是否能产生强烈共鸣 |
| 可执行性 | 15% | 在给定约束下是否可落地 |
| 差异性 | 10% | 与已有内容相比是否有新意 |

**放大规则**：每个角色为自己评分最高的候选做 **30秒辩护**（一段话），同时指出自己评分最低的候选的致命缺陷。

## delegate_task 并行调用示例

本框架的3轮讨论中，每一轮内部的3个角色操作可以并行执行，但**轮次之间存在顺序依赖**（第二轮需要第一轮的结果，第三轮需要第二轮的结果）。使用 `delegate_task` 分3轮依次调用。

### 第一轮：发散（Diverge）

### 关键设计说明

1. **轮次间串行、轮内并行**：3轮之间严格顺序执行（第二轮依赖第一轮输出，第三轮依赖第二轮输出），但每轮内的3个角色可完全并行。
2. **前轮上下文传递**：通过变量 `{round1_xxx_candidates}` 和 `{synthesized_candidates}` 将前一轮结果注入下一轮 context，确保角色能看到前轮结论。
3. **主 Agent 职责**：轮次间的编排、候选合成（第二轮后）和最终汇总排序由主 Agent 完成，子任务只负责单角色的视角输出。
4. **变量替换**：所有 `{variable}` 占位符由主 Agent 在调用 delegate_task 前替换为实际内容。

## 常见问题

### Q: 3个角色能否合并为2个？
A: 不建议。3角色构成最小三角验证——主题深度（内容）× 传播力（渠道）× 读者共鸣（受众），缺一个维度就失去平衡。

### Q: 能否增加到5个角色？
A: 可以，但每增加1个角色，讨论复杂度指数级增长。建议先用3角色跑通，再根据需要增加（如"竞品分析专家""商业决策者"）。

### Q: 置信度低于50%怎么办？
A: 说明输入的主题描述可能过于宽泛或与目标受众不匹配。建议：① 收窄主题范围；② 重新定义目标受众；③ 补充更多已有素材。

---

## 常见陷阱

1. **角色同质化**：3个角色说着一样的话，没有真正碰撞。检查方法——如果主题专家和读者代表的发言能互换，说明角色没坐实。
2. **过早收敛**：第一轮就开始互相点评，扼杀了发散阶段的创意。严格按轮次执行。
3. **放大变投票**：角色只是说"我同意/不同意"，没有给出新论据或新角度。放大必须带来新信息。
4. **忽略淘汰原因**：只看Top3不看淘汰摘要。淘汰原因往往包含重要洞察。
5. **置信度虚高**：所有候选都在80%以上。真实情况下置信度分布应该在40%-85%之间，如果都高说明评估标准太松。

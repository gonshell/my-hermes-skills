# 技术声明的来源链审计方法

## 适用场景

当分析一个技术概念/产品/理论时，其关键技术数据（性能数字、密度、效率、预测值）在网络广泛传播，但来源链不清晰。**在报告任何具体数字之前，必须完成来源链审计。**

典型触发信号：
- 用户问"XX 是什么"，而 XX 是最近的热门技术概念
- 用户要求"专业/学术角度分析"
- 用户追问"可行性、待验证、数据来源都有么？"
- 技术数据在中文自媒体大量传播（"刷屏全网""让XX沉默"）

## 五步审计流程

### Step 1：建立来源层级表

将所有可找到的信息源按可信度分层：

| 层级 | 来源类型 | 示例 | 含具体数字？ |
|------|---------|------|-------------|
| ① 一手-官方 | 企业官网新闻稿、官方博客 | huawei.com 新闻稿 | 通常只有定性描述 |
| ② 一手-学术 | 会议演讲、预印本论文、期刊论文 | ISCAS keynote、ChinaXiv/arXiv | 是 |
| ③ 二手-转载 | 技术社区转载（腾讯云、InfoQ） | cloud.tencent.com 文章 | 可能有转述偏差 |
| ④ N 手-自媒体 | CSDN、公众号、博客 | 各种"深度解析" | ❌ 不可作为依据 |

### Step 2：定位每个数字的最早出处

对报告中引用的每个关键技术数字，追溯其最早出现的位置：

```
数字 X
  → 在哪些文章中出现？
    → 这些文章引用了谁？
      → 最终追溯到哪个一手来源？
```

**关键检查点**：
- 官方新闻稿是否包含这个数字？（很多情况下**不包含**——企业公关稿刻意只放定性描述，数字仅在预印本/演讲中出现）
- 如果数字只出现在预印本中，检查预印本平台的性质

### Step 3：验证学术来源的评审状态

| 平台 | 性质 | 可信度 |
|------|------|--------|
| IEEE 期刊（JSSC、TCAS-I） | 同行评审 | 最高 |
| 顶级会议（ISSCC、VLSI Symposia） | 同行评审 | 最高 |
| arXiv / ChinaXiv | **预印本，未经同行评审** | 中——作者自报，未经独立验证 |
| 企业自发布白皮书 | 企业声明 | 中——需独立验证 |

> **ChinaXiv 特别说明**：中科院运营，对标 arXiv。首页明确声明"未经严格的同行评审"。任何只在 ChinaXiv 出现的数据，应标注为"作者声称，未经独立验证"。

### Step 4：检查传播偏差

同一个数字在不同传播层级的版本是否一致？

**真实案例（2026-06-14 韬定律审计）**：
- 论文原文：晶体管密度 155 → 238 MTr/mm²（计算增幅 53.5%）
- 腾讯云转载：写"55%"（四舍五入夸大）
- CSDN 博客：写"53.5%"（准确但来源标注缺失）
- 自媒体：直接写"密度提升超50%"（模糊化）

→ 在报告中应使用论文原文的计算值，并标注传播中出现的偏差版本。

### Step 5：逐项数据审计表

对每个关键数据，输出以下审计结果：

| 维度 | 填写内容 |
|------|---------|
| **来源层级** | ① 官方稿 / ② 预印本 / ③ 转载 / ④ 自媒体 |
| **可验证性** | 外部能否独立核实？如何核实？ |
| **对照基准** | 行业公开可比数据（用于判断数字合理性） |
| **传播偏差** | 是否有多个版本？偏差方向？ |
| **综合可信度** | 声明可信 / 待验证 / 无法确认 |

## 输出格式

来源链审计应作为报告的独立章节（通常放在技术分析之后、结论之前），包含：

1. **信息来源链路表**（层级 + 性质 + 是否含数字）
2. **关键平台性质说明**（如 ChinaXiv 预印本未经评审）
3. **逐项数据审计表**（每个数字一行）
4. **传播偏差实证**（同一数字的不同版本对比）
5. **第三方独立评价**（行业分析师、研究机构的原文引用）

## 与现有标注体系的关系

来源链审计是对 [N/T/E] 标注体系的**前置工作**——先审计来源链，才能正确标注 [N/T/E] 和置信度级别。

- 来源链审计后确认为"官方 Datasheet"→ 标注 [N] + [FACT]
- 来源链审计后确认为"仅预印本，无第三方验证"→ 标注 [N]（自报）+ 降级置信度
- 来源链审计后发现传播偏差 → 标注原始值 + 传播偏差说明

## 本方法论的来源

2026-06-14 韬(τ)定律技术分析 session。用户追问"可行性、待验证、数据来源都有么？你要严谨的论述"后，发现所有中文网络讨论引用的具体数字（53.5% 密度提升、41% 能效、α=10×/年）全部追溯到同一篇 ChinaXiv 预印本（未经同行评审），华为官方新闻稿刻意不含任何精确数字，第三方分析机构认可方向但拒绝确认数据。中文网络形成了"信息回音壁"——看起来证据充分，实际只有一个未经审查的来源。

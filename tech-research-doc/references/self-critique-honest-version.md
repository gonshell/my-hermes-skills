# 研究报告自我批评与诚实版改写模式

> 新增于 2026-06-21 session。源案例：宇通客车估值分析反思与重写。

## 适用场景

用户对已交付的研究/分析报告提出**元层面**的批评时：

- "上述的内容客观么，是实事么，评分标准是什么？"
- "你深刻反思：方法、公式、数据真实、合理吗？"
- "这个分析是伪科学吧？"
- "把评分系统说清楚"
- "评估/估值/护城河分析的依据是什么？"

**核心信号**：用户不是在质疑**结论**错没错，而是在质疑**方法论本身**是否站得住。

## ⚠️ 第一原则：识别"伪科学陷阱"

**这是本 session 最关键的教训。** 当用户说"反思"时，往往意味着报告**已经掉进了"用精确形式包装主观判断"的陷阱**。

**伪科学陷阱的 5 个特征**：

| 特征 | 表现 | 案例（本 session） |
|---|---|---|
| 1. 评分无锚定标准 | "9/10、8/10"但没定义 9 分对应什么客观阈值 | 护城河评分 9/10、8/10 |
| 2. 概率是叙事概率 | 6 情景概率 10%/20%/30% 完全主观编造 | 估值 6 情景概率 |
| 3. 数字估算冒充披露 | EBITDA 是用营业利润 + 折旧摊销估算，但报告里说"是 XX 亿" | 2025 EBITDA ≈ 71 亿（估算） |
| 4. 多方法平均 = 客观 | 把不同假设的估值简单加权平均 | PE + DDM + EV/EBITDA 取均值 |
| 5. 接受营销话术 | 把"行业首个""国内独一档"当客观事实 | "睿控 E+ 是行业首个" |

**判定方法**：把每个数字、每个评分、每个概率都问一句"这个的客观依据是什么？"——如果答不上来，就是伪科学成分。

## 标准动作链（不要直接动笔）

### 步骤 1：先承认问题

**不要**：狡辩、找借口、强调"但大部分是对的"。

**要**：明确指出报告中的哪些具体表述有问题（哪几个评分、哪几个数字、哪几个方法）。

```
反例：
"本报告的评分系统确实有改进空间..."

正例：
"上版报告用了 9/10、8/10 的评分，但 9 分对应的客观阈值我没说。
这本质上是用'精确形式'包装'主观印象'。下面是我用文字承认的失败：
1. 评分无锚定标准
2. 概率完全是主观的
3. EBITDA 是估算的
4. 多方法简单加权无意义
5. 我接受了营销话术"
```

### 步骤 2：拆分"事实 / 判断 / 预测"

每段写完后问一句"这段是事实/分析/预测/洞见哪一类"——`tech-research-doc` SKILL.md 已有这个分类。

**诚实版的判定标准**：
- ✅ 事实（已披露数据）→ 可以保留
- ✅ 分析（基于事实的推理）→ 需要标注推理链
- ⚠️ 预测（带条件）→ 必须有置信度 + 反方
- ❌ 没有依据的"我觉得" → 删除

### 步骤 3：诚实版重写

**不要修改原文件**——保留作为"伪科学反面教材"。

**生成新文件**：`<原文件名>_诚实版.md`

```
原文件：宇通客车_估值分析.md        ← 保留（作为反面教材）
新文件：宇通客车_估值分析_诚实版.md  ← 重写（用区间代替目标价）
```

### 步骤 4：诚实版的核心修改

| 原版（伪科学） | 诚实版 | 修正理由 |
|---|---|---|
| "目标价 40 元" | "估值参考区间 25-40 元（不确定性 ±30%）" | 承认不确定性 |
| "上行空间 40-55%" | "距离中位 31-37 元有 20-40% 空间" | 区分估值空间和预测 |
| "中性目标价 36.4 元" | "加权区间 31-37 元" | 用区间代替点估值 |
| "概率加权 40.5 元" | "加权区间 31-37 元" | 强调区间宽度 |
| "83% 方向准确率" | 删除（6 样本无统计意义） | 避免伪统计 |
| "本报告可作为投资参考" | "本报告不构成投资建议" | 明确免责 |

### 步骤 5：明确说明估值的不确定性

**任何股票估值/分析都有 ±30% 的不确定性**——这是常识。诚实版必须**显式声明**这个区间。

**模板**：
```
本分析的核心局限：
1. 样本不足——6 个历史数据点无法给出统计显著的回测
2. 方法局限——多数方法不适用本研究对象
3. 数据估算——XX 是估算而非披露
4. 概率主观——情景概率是叙事概率不是统计概率
5. 预测不可能——任何"目标价"都是估值不是预测
```

## 典型陷阱

### 陷阱 1：用"但大部分是事实"逃避批评

**错误做法**：
"虽然评分标准没定义，但大部分数据是真实的。"

**正确做法**：
"评分标准未定义 = 整个评分系统无效。删除所有数字评分。"

### 陷阱 2：诚实地承认问题，但用"需要继续研究"逃避重写

**错误做法**：
"我承认评分有问题，但这需要更多研究才能给出更好的版本。"

**正确做法**：
直接重写并生成诚实版文件。**用户要的不是承认问题，是修订后的版本**。

### 陷阱 3：诚实版 = 减法（删掉主观部分）但保留形式

**错误做法**：
保留"目标价 40 元"但加个脚注"仅供参考，不构成投资建议"。

**正确做法**：
"目标价 40 元" → "估值参考区间 25-40 元（不确定性 ±30%）"。**改变结构**而不只是加免责。

### 陷阱 4：诚实版 = 加上"我们假设"标签但保留原结论

**错误做法**：
"假设分红率持续 99%+，我们给出目标价 40 元。"

**正确做法**：
"分红率 99.65% 是否可持续是核心风险。如果分红率回落到 60-70%，DDM 估值会腰斩至 20 元以下。**这是最大的被低估风险。**"

## 触发信号识别表

| 用户表述 | 触发的反思模式 |
|---|---|
| "反思一下" / "深刻反思" | 全部内容都需要元层面审查 |
| "客观么" / "是实事么" | 数据来源 + 方法论审查 |
| "评分标准" | 任何打分/评级系统需要明确锚定 |
| "科学有依据" | 方法适用性 + 公式前提 + 数据可靠度 |
| "再来" / "重写" | 用诚实版替换，不只是修改 |
| "承认吧" | 用户已经判断有问题，需要直接承认 |

## 与"反思+出方案+等确认"协议的区别

| 协议 | 触发场景 | 关键动作 |
|---|---|---|
| `reflect-then-confirm` | 任务开始前 | 反思设计意图，出方案，等用户确认 |
| **本模式（self-critique-honest-version）** | 报告交付后被质疑 | 承认问题，分类问题（事实/判断/预测），重写诚实版 |

**关系**：本模式是 `reflect-then-confirm` 协议的事后版本——任务开始前用前者，任务被质疑后用本模式。

## 质量检查清单

诚实版完成前，逐项检查：

- [ ] 每个数字标注了来源（年报/季报/估算/不知道）
- [ ] 每个方法说明了适用性（适用/部分适用/不适用）
- [ ] 每个概率标注了"叙事概率"还是"统计概率"
- [ ] 估值给区间不给点（25-40 元而不是 40 元）
- [ ] 显式声明不确定性（如 ±30%）
- [ ] 区分"事实"和"判断"
- [ ] 每个判断配反方观点
- [ ] 显式说明"不构成投资建议"或"不是预测"
- [ ] 原文件保留作为对比（不要删除）

| 原版错误 | 诚实版修正 |
|---|---|
| "目标价 40 元" | "估值参考区间 25-40 元" |
| "上行空间 40-55%" | "距离中位 31-37 元有 20-40% 空间" |
| "概率加权 40.5 元" | "加权区间 31-37 元" |
| "中性目标价 36.4 元" | "加权区间 31-37 元" |
| "DDM 法测算价 36 元" | "DDM 法不适用此股（数学上失效），仅作参考" |
| "83% 方向准确率" | "6 个样本无统计意义，删除此结论" |
| "期望收益 +7.5%" | "期望收益 +7.5%（基于主观情景概率，置信度低）" |

## 经验教训

1. **诚实版不是示弱，是产品迭代**：用户能识别伪科学是好事，承认 + 重写比狡辩更专业
2. **保留原文件是诚实的标志**：不删除"出错版本"显示对自己的反思有底气
3. **诚实版的"诚实"在于结构变化**：不只是加免责，是改报告结构本身
4. **估值/分析的局限性是常识**：任何量化分析都有不确定性，承认这一点不丢人
5. **诚实版比原版更有用**：用户会优先使用诚实版作为决策依据

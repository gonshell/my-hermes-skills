# 个股走势复盘报告模板

> 独立于 6 种报告类型。新增于 2026-06-21 session。源案例：宇通客车（600066.SH）半年走势复盘。

## ⚠️ 会话前置：先反思+出方案+等确认（feishu-content-analysis 教训 0 协议）

**触发信号**（任一即触发本协议）：
- 用户问"分析 XX 股票" / "复盘 XX 走势" / "XX 涨跌归因" / "XX 半年/季度/年走势"
- 用户在已有报告上要求"重做" / "反思" / "深度思考"
- 用户对之前的产出表达不满（"不对" / "再来" / "方向不对"）

**标准动作链**（不要直接动笔）：

1. **先问用户 3 个问题**（不主动展开分析）：
   - 时间窗口：半年 / 季度 / 一年 / 自定义？
   - 分析目的：投资决策 / 内部研究 / 写报告 / 验证假设？
   - 输出深度：标准（8-10 节） / 加深（10-15 节含子章节） / 战略级（20+ 节含完整时间表）？
2. **出结构方案**（按"标准结构"或"最小可行结构"列出目录）
3. **列出假设与边界**：数据截止日 / 数据源 / 覆盖范围
4. **停下等用户确认**——不要直接动手

**反例（历史失败）**：
- 用户问"分析 XX 股票" → 直接写 6 章节完整报告 → 用户反馈"方向不对，要的是 X 不是 Y"
- 正确做法：先 3 个问题 + 结构方案 + 等确认 → 用户确认后再写

**为什么必须遵守**：
- 股票走势复盘是高风险研究任务，结论会用于投资/竞争决策
- 不同时间窗口（半年 vs 季度）的分析方法差异大
- 没有数据截止日，结论会过时

**详见**：feishu-content-analysis skill 教训 0（最完整的方法论）。本协议适用于所有研究类任务，不限于本模板。

## 适用场景

用户给出一只股票 + 一个时间窗口（季度/半年/年），要求做走势复盘 + 驱动力拆解。读者是想理解"这段时间这只股票发生了什么 / 为什么 / 接下来怎么看"的投资者或研究员。

**与 6 种报告类型的关系**：

| 现有类型 | 能否套用 | 不能套的原因 |
|---|---|---|
| 状态评估 | 形似神不似 | 状态评估是"现在处于什么水平"（横向对标），复盘是"过去这段时间发生了什么"（纵向归因） |
| 趋势研判 | 形似神不似 | 趋势研判是"未来往哪走"（前瞻），复盘是"过去为什么这么走"（回顾） |
| 深度分析 | 部分可借 | 深度分析的"推理模型 → 逐层分析 → 替代解释"可借，但缺驱动力拆解结构 |

**判断标准**：文档是否在回答"过去 X 时间内，这只股票的 Y% 涨跌来自哪些驱动因素"？是 → 走本模板。不是 → 走 6 种类型。

## 标准结构

```
1. 执行摘要（必选）
   - 一段话：时间段、起止价格、累计涨跌幅、核心驱动一句话
2. 时间窗口与基础数据（必选）
   - 起点/终点价格、市值、估值水位
   - 同期大盘/行业基准对比
3. 财务面（必选）
   - 期内 N 期财报（季报/中报/年报）
   - 关键指标：营收、净利、毛利率、净利率、现金流
   - 同比 vs 环比
4. 业务面（必选）
   - 月度/周度销量数据
   - 区域/产品/客户结构变化
   - 公开订单/中标/合同清单
5. 公开事件清单（必选）
   - 公告（财报、分红、回购、股权变动、监管）
   - 行业政策（补贴、监管、标准）
   - 重大事件（并购、纠纷、人事）
   - 卖方研报与一致预期
6. 上涨动力（必选）⚠️ 核心章节
   - 分 2-3 层（如：故事层 / 兑现层 / 催化层）
   - 每层：时点 + 关键数据 + 市场逻辑
7. 下跌动力（必选）⚠️ 核心章节
   - 分 2-3 层（与上涨动力对称）
   - 每层：时点 + 关键数据 + 市场逻辑
8. 估值迁移路径（必选）⚠️ 核心章节
   - 期内隐含 PE 起点 / 高点 / 低点
   - 业绩贡献 vs 估值回归的归因拆解
9. 关键判断与拐点信号（必选）
   - 接下来关键观察点（哪些数据/事件能验证或证伪当前判断）
10. 风险与机会（必选）
    - 对称列出
11. 替代假设（必选）
    - 如果主判断错了，最可能的替代解释是什么
    - 每个替代假设附验证方法
12. 信息源（必选）
    - 行情数据、财务报告、产销数据、IR 记录、卖方研报
13. AI 能力边界声明（必选）
    ⚠️ 必须声明：所有判断基于公开信息 + 公告时点数据，不构成投资建议
```

## 最小可行结构（时间有限时）

```
执行摘要 → 时间窗口基础数据 → 财务+业务+事件 三表合一 → 上涨/下跌动力（各 2-3 层）→ 关键判断
```

## 质量检查清单

- [ ] 上涨动力 / 下跌动力 是否对称拆解（不是只讲"为什么跌"）
- [ ] 估值迁移是否单独成节（不能藏在股价骨架里）
- [ ] 每个动力层是否都有"时点 + 关键数据 + 市场逻辑"三要素
- [ ] 月度销量 / 季报数据是否有同比和环比两个维度
- [ ] 公开事件是否区分了"公司公告 / 行业政策 / 卖方研报"三类
- [ ] 替代假设是否给出了可验证方法（不是空话）
- [ ] AI 能力边界声明特别强调"不构成投资建议"
- [ ] 数据截止日期明确（"截至 YYYY-MM-DD，含 X 月 X 日收盘"）

## 核心方法论：上涨动力 + 下跌动力 对称拆解

**为什么必须对称**：半年/季度报告如果只讲"为什么跌"，读者无法理解"为什么之前会涨那么高"；反过来同理。两端都讲清楚，才能让"为什么波动这么大"这个问题有答案。

**拆解框架**（来自本 session 实战）：

### 上涨动力的三层结构（通用模板）

| 层次 | 性质 | 驱动来源 | 典型数据 |
|---|---|---|---|
| **故事层** | 长周期叙事 | 低基数 + 新业务突破 | 同比 +100% 以上的初期爆发 |
| **兑现层** | 中期业绩验证 | 业绩持续超预期 | 同比 +30%~+50% 的稳定增长 |
| **催化层** | 短期情绪 | 公告 / 分红 / 政策 / 概念 | 单日 +10% 涨停、突破历史新高 |

三层强度递减：故事层提供"未来 3 年能涨"的想象空间，兑现层提供"未来 1 年能涨"的逻辑，催化层提供"未来 1 个月能涨"的资金。**真正把估值推到极端的是催化层**（往往是最后一棒），后续下跌的伏笔也埋在催化层。

### 下跌动力的三层结构（通用模板）

| 层次 | 性质 | 驱动来源 | 典型数据 |
|---|---|---|---|
| **业绩兑现失败** | 短期业绩打击 | 营收/利润首次同比转负 | Q1 净利 -10%~-15% |
| **叙事动摇** | 中期叙事质疑 | 高增长可持续性 / 监管 / 竞争 | 增速边际放缓、机构下调评级 |
| **估值消化** | 结构性回归 | 前期涨幅过大（2-3 年 +200%+） | PE 从 25 倍回归到 12-15 倍 |

三层权重通常是：业绩兑现失败 > 估值消化 > 叙事动摇。**估值消化与业绩无关**——即使业绩持平而非下滑，前期涨幅过大也足以触发杀估值。

## 关键技巧：估值迁移归因

半年涨跌不能只看"业绩贡献多少"，必须拆出"估值贡献多少"。

**简单归因法**：

```
股价变化 ≈ EPS 变化 × 估值倍数变化

例：某股半年 -19%
  → 业绩贡献：净利 -12.69%（单变量）→ 大约 -10% ~ -12% 股价
  → 估值贡献：PE 25 倍 → 12-13 倍（-50%）→ 大约 -8% ~ -10% 股价
  → 加合 ≈ -19%，与实际接近
```

**为什么必须做这个拆解**：用户最关心的是"业绩好不好"+"估值贵不贵"两个独立维度。如果只讲业绩，会让读者误以为"业绩差所以跌"——而实际可能是估值消化主导，业绩其实没怎么恶化。

## 数据收集的优先级

按可信度从高到低：

```
① 公司公告（上交所披露的财报、月报、临时公告）
   └── 最可信，所有结论的支撑

② 投资者关系活动记录表
   └── 管理层口径，最直接的态度信号

③ 卖方研报（≥10 家覆盖的口径）
   └── 一致预期、市场情绪、行业观点

④ 财经媒体（财联社、第一财经、上海证券报等）
   └── 用于补充公告时点和上下文

⑤ 行情数据（中财网/同花顺/东方财富/搜狐证券）
   └── 价格/成交量/资金流向
```

**不要用**：
- 自媒体未署名文章
- 论坛/雪球单帖
- 未经核实的"小道消息"

## 关键技巧：与已有报告融合时的一表审计

当用户要求"把新分析和已有报告融合"时（不是推倒重写），按以下四档做一次审计，再决定怎么改：

| 标记 | 含义 | 处置 |
|---|---|---|
| ✅ 一致 | 新旧数据/结论完全吻合 | 保留 |
| ⚠️ 未覆盖 | 新分析有，旧报告无 | 新增章节 |
| ❌ 冲突 | 新旧结论矛盾 | 标注冲突点，向用户确认 |
| ➕ 增量 | 旧报告有但被新分析扩展 | 在原章节内补充 |

**为什么这样做**：避免"为了改而改"——只新增和修订必要的部分，保持报告整体结构稳定。用户的偏好是"patch，不要 rewrite"。

## 典型陷阱

1. **把"事件"当"原因"**——某日某公司发布某公告 ≠ 该公告导致了股价变动。需要对比公告前后股价反应、量能变化、是否同步发布。
2. **只讲跌幅不讲涨幅**——半年报告最容易踩这个坑。读者需要知道"为什么之前能涨到 38 元"才能理解"为什么现在跌到 26 元"。
3. **把同比和环比混用**——同比是"和去年同期比"，环比是"和上一期比"。月报数据用同比（季节因素），季报数据用同比 + 环比。
4. **海外/国内结构混淆**——很多公司国内国外业务结构差异大，需要分两条线分析，不能合并。
5. **估值倍数口径不一致**——PE-TTM 和 PE-Forward 数值可能差 30%+，跨期对比必须用同一口径。

## 与 feishu-content-analysis skill 的边界

| 维度 | tech-research-doc（本文） | feishu-content-analysis |
|---|---|---|
| 输入 | 公开市场数据 + 公司公告 | 飞书 doc（视频清单 / 资讯清单） |
| 输出 | 走势复盘 + 驱动力 + 关键判断 | 跨源综合分析 + 调研建议 |
| 核心方法 | 上涨/下跌动力对称 + 估值迁移 | 事件流 + 判断流 / 三层金字塔 |
| 触发词 | "分析 XX 股票"、"复盘 XX 走势" | "分析这个飞书 doc"、"综合解读" |

两者都强调"不要臆测，做统计/分析/预测/洞见"，但数据源和分析结构不同。**不要互相套模板**。

# AI 能力边界声明模板

## 为什么必须声明

AI 生成的研究报告存在系统性盲区：无法实验验证、无法获取一手数据、无法做专家级判断。不声明这些边界，会误导读者把 AI 推理当成专家结论。

---

## 通用声明模板（所有报告类型必须包含）

```markdown
## AI 研究能力边界声明

本报告由 AI 辅助生成。以下能力边界适用于全文所有结论：

### ✅ 本报告中 AI 执行的工作
- 搜索并整合公开信息（官方文档、学术论文、开源仓库、行业报告）
- 基于多源数据做结构化分析和逻辑推理
- 交叉验证不同来源的信息一致性
- 生成结构化报告框架和可视化方案

### ❌ 本报告中 AI 未执行的工作
- 实验验证：未运行任何 benchmark、未实测任何性能数据
- 定量建模：未建立精确的数学/统计模型
- 专家判断：未咨询领域专家，所有分析基于公开信息
- 一手调研：未进行用户访谈、未获取企业内部数据

### 📊 标注体系说明
- `[N]` 官方数据 · `[T]` 行业经验 · `[E]` 估算 → 标注数据来源
- `[FACT]` 已验证事实 · `[HCF]` 高置信推断 · `[REASON]` 合理推测 · `[TREND]` 趋势外推 → 标注判断可信度
- 本报告最高使用到 [XXX] 级别

### ⚠️ 建议读者
- 将本报告视为「研究起点」而非「最终结论」
- 关键决策前，建议补充：实地测试、专家评审、一手数据验证
```

## 按报告类型的差异化声明

### 技术调研 / 竞品对比 / 状态评估

追加声明：

```
本报告的结论基于公开可获取的信息。[HCF] 级别的推断已用 ≥2 个独立信源交叉验证，
但仍可能存在：信源过时（技术迭代快）、信源偏差（厂商数据偏乐观）、缺失信息（非公开数据）。
建议在采购/选型决策前，以实际 PoC 测试结果为准。
```

### 深度分析

追加声明：

```
本报告包含 [REASON] 级别的推理——即基于已知事实的逻辑推演，但缺少直接实验验证。
推理链的每一步已标注前提条件，读者可自行判断前提是否成立。
如果任何前提被证伪，对应结论需要重新评估。
```

### 趋势研判

追加声明：

```
⚠️ 趋势预测的固有局限性：
- 本报告使用 [TREND] 级别的外推，基于历史数据的线性/非线性延伸
- 技术发展存在拐点和黑天鹅事件（如突破性论文、政策突变、地缘冲突），无法预测
- 所有预测已标注时间范围和核心假设，假设变化时预测需更新
- 预测不应作为投资或战略决策的唯一依据
```

### 方案设计

追加声明：

```
⚠️ 本方案未经实际验证：
- 架构设计基于技术调研和逻辑推理，未经过 PoC 验证
- 工期估算为 [E] 级别，实际工期受团队能力、环境复杂度影响，偏差可能达 ±50%
- 强烈建议在全面实施前，先完成小规模 PoC 验证核心假设
- 风险缓解措施的有效性未经测试
```

## 声明放置位置

| 位置 | 适用场景 |
|------|---------|
| 文末独立章节 | 默认位置，所有类型适用 |
| 执行摘要末尾 | 报告较长（>5000字）时，在摘要中加一句提醒 |
| 每个核心结论后内联 | 预测类内容（趋势研判）和高风险决策（方案设计） |

### 示例：趋势研判的内联声明

```
预计 2027 年国产 AI 芯片市场份额可达 15-20% [TREND]

> ⚠️ 此为趋势外推，假设条件见第4章。若出口管制政策放宽或出现技术突破，
> 此预测需重新评估。建议每季度审视一次假设是否仍然成立。
```

## 常见问题

**Q：声明会不会削弱报告的可信度？**
A：相反。不声明的报告一旦被挑出未验证结论，整篇可信度归零。声明反而建立信任——读者知道哪些可以放心用、哪些需要自己验证。

**Q：如果用户要求去掉声明呢？**
A：提醒用户风险，但尊重用户决定。在报告中保留一句话："本报告经过人工审核确认"（如果确实有审核）。

**Q：报告全部是 [FACT][N] 级别，还需要声明吗？**
A：需要。即使数据全部来自官方来源，AI 的**筛选、组织、分析**过程仍可能引入偏差（选择性地呈现了某些数据而忽略了另一些）。

# 报告评审方法论 — 对已有报告的质量审查

本文档为**报告评审**任务提供方法论指导，作为 tech-research-doc 的补充。当用户给出一份已有报告并要求"核查正确性"、"评估结构"、"判断覆盖完整性"时，使用本方法论。

---

## 评审任务类型识别

当用户说这些话时，触发**报告评审**任务：

- "请审查这份报告的正确性"
- "请评估这个调研的目录结构是否合适"
- "这份报告是否覆盖了大家讨论的问题"
- "请核查数据是否准确"
- "读完后请确认内容是否合适"

**不是创作任务**：不执行四阶段写作流程，而是对已有内容的元评估。

---

## 评审维度框架

### 维度一：数据正确性核查

**方法：多源交叉验证**

```
对于报告中的每个关键数据点：
  ① 记录原文数据（保留引用）
  ② 搜索权威来源（优先顺序）：
     A. 政府/官方机构数据（国家市监总局、中关村协会、欧盟统计局等）
     B. 权威研究报告（亿欧智库、IDC、Gartner等）
     C. 学术论文
     D. 媒体报道（辅助参考）
  ③ 对比差异
  ④ 分类输出：
     ✅ 可信（有权威来源支撑，数字接近）
     ⚠️ 偏差（数字有差异，需说明）
     ❌ 错误（方向相反或数量级差异）
```

**常见数据核查优先级**（以OPC报告为例）：

| 数据类型 | 优先验证来源 |
|---------|------------|
| 中国OPC规模/增速 | 中关村人才协会报告、国家市监总局报告、新华网 |
| 政策文件内容 | 政府官网原文（.gov.cn）、政府公众号 |
| 全球对比数据 | 欧盟统计局（欧盟）、MetaEra/腾讯新闻（美国） |
| 融资/股权数据 | 媒体交叉验证、海盈云等财经来源 |
| 企业案例 | 企业官方公告、创始人公开访谈 |

**⚠️ 已知易错点**：
- 美国OPC数量（真实≈2980万，报告常写3040万）
- 欧盟OPC数量（真实≈1800万，报告常低估至1200万）
- 政策文件措辞（"支持措施"≠"法律定义"，切忌夸大）
- 社区数量（"数十个"是常态，"143个"需找到精确统计来源）

### 维度二：结构评估

**方法：对照报告类型模板**

```
1. 识别报告主类型（技术调研/深度分析/趋势研判/竞品对比/方案设计/状态评估）
2. 对照对应类型的标准结构模板
3. 检查：
   - 必选章节是否都存在
   - 是否有不应该混合进来的章节（如需求分析中混入技术选型）
   - 章节顺序是否符合该类型的逻辑流
4. 输出缺失章节和多余章节清单
```

**OPC研究报告的特殊性**：这是一份**混合型**报告（发展趋势+行业分析+商业机会），主类型是**趋势研判**，附加模块参考**行业分析**框架。

### 维度三：内容覆盖评估

**方法：热点矩阵 + 差距分析**

```
1. 识别该主题的公众讨论热点（通过网络搜索抓取讨论集中点）
2. 将报告章节与热点逐一匹配
3. 输出覆盖矩阵：
   | 讨论热点 | 报告是否覆盖 | 覆盖深度 |
   |---------|------------|---------|
   | XXX     | ✅ 完整     | 详细     |
   | YYY     | ⚠️ 浅层     | 只有标题 |
   | ZZZ     | ❌ 缺失     | —        |
4. 识别明显缺失且重要的热点
```

### 维度四：总体评分（结构化输出）

| 维度 | 评分 | 说明 |
|------|------|------|
| 数据可信度 | X/100 | 主要数据基本准确，误差比例 |
| 结构完整性 | X/100 | 框架完整但某章节缺失 |
| 覆盖深度 | X/100 | 宽度够但多处浅尝辄止 |
| 时效性 | X/100 | 信息是否较新 |
| 实用价值 | X/100 | 偏宏观叙事还是指导实践 |

## 执行原则

1. **先查数据，后评结构，再评覆盖** — 数据错误比结构问题更严重
2. **不重写，只评估** — 评审不等于代笔，给出结论而非重新创作
3. **标注来源** — 每个判断都要有依据
4. **区分"报告自身质量"和"主题本身复杂性"** — 报告可能已完整但主题本身无解，不要给报告打低分
5. **平衡正面和负面** — 既指出问题也认可做得好的地方

## AI 能力边界在评审中的体现

评审类任务中 AI 的局限：

| AI 能做 | AI 不能做 |
|---------|----------|
| 多源数据对比 | 一手验证（实地采访、实验测试） |
| 逻辑一致性检查 | 专家级领域判断 |
| 公开来源的查证 | 非公开内部数据的获取 |
| 结构化评估框架 | 对未公开信息的了解 |

**声明方式**：在评审报告末尾加一行：
> ⚠️ 本评审基于公开信息和逻辑推理，非实地调研。关键数据建议读者自行向来源方求证。

# 洞见模板 v2 — 反常断言 + 涌现式维度 + 倾向 + 替代假设

> 教训 21 的来源上下文。洞见模板从 v1（标签式）→ v2（3 段式）→ v3（本版）的演进记录。
> v3 版本已于 2026-06-14 被用户确认并批量执行。

## 演进历程

| 版本 | 结构 | 被否原因 |
|---|---|---|
| v1 | 标签式（"洞见 N：标题"） | "每条洞见想说什么？"——标签是空壳 |
| v2 | 3 段式（现象 / 技术 / 产品 / 商业 / 未验证） | "AI 味太足"——3 段平衡 = 套路 |
| **v3** | 反常断言 / 涌现式维度 / 倾向 / 替代假设 | **用户确认批量执行** |

## v3 模板

```
现象：[1-2 句反常描述 + 判断钩子，不用数据罗列]
可能的解释（维度名从现象涌现，不预设 技术/产品/商业）：
- [维度名即问题]：[解释]
- [维度名即问题]：[解释]
倾向：[最相信哪个解释 + 一句理由]
未验证 / 替代假设：[如果洞见是错的，最可能的替代解释 + 怎么区分]
```

## 4 个改进点的规则

### 改进 A：现象段 = 反常断言

规则：用 1-2 句话描述一个具体的反常，不用聚合数字。最后一句必须是判断钩子。

❌ "B 站 22 条总播放 1.07 亿，是晚间档 899 万的 11.9 倍。"
✅ "B 站排名第一的不是 AI 新闻，是搬运 MIT 课程的账号。高播放 ≠ 热点，高播放 = 补课。"

### 改进 B：维度名从现象涌现

规则：不预设"技术/产品/商业"三维。维度命名是具体判断句。

❌ "技术：LLM 把 Prompt+Agent 变成新基础能力"
✅ "为什么是'搬运+中文字幕'而不是原创：中文技术圈对英文教学资源的依赖是结构性的"

### 改进 C：加"倾向"行

规则：标明最相信哪个解释 + 一句理由。是立场标记，不是结论。

"倾向：更相信'结构性依赖'——如果只是补课需求，不会集中出现'搬运+重组'这种特定形态。"

### 改进 D：未验证升级为"替代假设"

规则：至少给一个替代假设 + 怎么区分两种解释的验证方法。

❌ "需 UP 主历史数据 + B 站推荐机制透明度"
✅ "替代解释是'算法在推课不是用户在补课'。区分方法：看完播率（主动补课→高完播；算法推流→低完播）。"

## 批量重构操作技巧

用 `execute_code` 批量 block_replace + block_delete：

```python
from hermes_tools import terminal

DOC = "<doc_token>"

insights = [
    ("insight1.xml", "标题block_id", "旧子block1,旧子block2,..."),
    # ... 每条洞见一组
]

for xml_file, replace_id, delete_ids in insights:
    # Step 1: replace 标题block → 完整新洞见内容（标题+段落+列表）
    terminal(f"cd /Users/xiesg/workspace && lark-cli docs +update --api-version v2 "
             f"--doc \"{DOC}\" --command block_replace "
             f"--block-id \"{replace_id}\" --content @./{xml_file}", timeout=30)
    # Step 2: delete 旧子blocks
    terminal(f"cd /Users/xiesg/workspace && lark-cli docs +update --api-version v2 "
             f"--doc \"{DOC}\" --command block_delete "
             f"--block-id \"{delete_ids}\"", timeout=30)
```

关键点：`block_replace` 的 `--content` 可以包含多个 block（标题 + 段落 + 列表），它会替换目标 block 为全部新内容。每条洞见的 XML 写在独立文件里。

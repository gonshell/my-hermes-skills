# 调研建议模板 — 从话题清单到可执行调研简报

> 教训 22 的来源上下文。调研建议从"标题列表+交叉引用"升级为"5 字段微型调研简报"。
> 已于 2026-06-14 被用户确认并批量执行。

## 旧模板问题

```
1. **话题标题** → 对应洞见 N（§x.x）
```

三个缺陷：
1. **是话题不是指令**——读完不知道第一步做什么
2. **不区分类型**——无法判断该用什么方法（读论文 vs 爬数据 vs 读财报）
3. **引用方向反**——指向已分析完的洞见，而不是洞见留下的未验证假设

## v2 模板

```
### 建议 N：[一句核心问题，不是话题标题]

类型：技术 / 市场 / 商业 / 政策 / 战略分析
为什么现在做：[时间窗口 + 不做会错过什么]
切入路径：
  1. [第一步：具体动作 + 数据源]
  2. [第二步]
  3. [第三步]
预期产出：[查完手里有什么——一个判断？一个数据集？一个框架？]
来源锚点：洞见 N（§x.x）的未验证假设 [一句话复述]
```

## 字段设计理由

| 字段 | 为什么必须有 |
|---|---|
| 核心问题（不是标题） | 把"话题"变成"问题"。问题有答案，话题没有。 |
| 类型 | 决定方法（读论文 vs 爬数据 vs 读财报 vs 读法规） |
| 为什么现在做 | 替代"评估口径"。标时间窗口和紧迫性，安排优先级 |
| 切入路径 | 把"去调研"变成"第一步做 X" |
| 预期产出 | 开始前就知道"做完算完成"。防止调研发散 |
| 来源锚点 | 不是"对应洞见 N"，而是"洞见 N 的哪个未验证假设驱动了这个建议" |

## 调研建议分类

| 类型 | 方法 | 典型产出 |
|---|---|---|
| 技术 | 读论文 / 跑模型 / 对比 benchmark | 技术笔记 + 技术判断 |
| 市场 | 爬数据 / 趋势分析 / 竞品对比 | 判断 + 数据集 |
| 商业 | 读财报 / 价格追踪 / 竞争分析 | 追踪基线 + 走势判断 |
| 政策 | 读法规 / 找处罚案例 / 跨国对比 | 合规清单 + 政策方向判断 |
| 战略分析 | 框架分析 / 多源推理 | 可行性框架 + 对比矩阵 |

## 调研建议来源

两类：
1. **洞见延伸**（主）：来自洞见的未验证假设/替代假设 → 设计验证路径
2. **事实层直接学习需求**（补充）：来自事实综述中的技术/政策事件，不经过洞见环节

非洞见驱动示例：
```
### 建议 5：1M token 上下文 + 代码模型专项化的技术实现路径
类型：技术学习
来源锚点：§2.1 技术事实——非洞见驱动，来自技术事实层的直接学习需求
```

## 用户确认路径（2026-06-14）

1. 出诊断（5 个问题）+ 样板（建议 2）
2. 用户确认 3 个决策点：
   - 删除评估口径（是元信息，对读者无用）→ 融入每条"为什么现在做"
   - 允许加非洞见驱动的技术学习建议
   - 5 字段详细程度可以
3. 批量执行 6 条调研建议（block_replace 标题块 + block_delete 旧块）

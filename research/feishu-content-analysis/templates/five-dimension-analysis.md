# 五维综合分析报告模板

直接复用本模板，填写具体内容。

---

> **2026-06-14 v3 用户批准结构**（默认推荐）
>
> 任务：飞书视频文档的快日报综合分析
>
> 章节：
> 0. 方法论限制（如有抓取工具失败，必须含；1 段）
> 1. 趋势（用户原始维度 1）
> 2. 技术（用户原始维度 2）
> 3. 热点（用户原始维度 3）
> 4. 商业（用户原始维度 4）
> 5. 产品（用户原始维度 5）
> 6+. 扩展维度（agent 自行加，2-3 个为佳；例：安全治理 / 媒体覆盖 / 视频 ID 索引）
> 末. 1 句话总结（不超过 100 字）
>
> 目标长度：**≤ 8 KB**（v3 实测 7.5 KB = v2 的 46%，信息密度反而更高）
>
> 视频 ID 写法：**内联紧贴内容** `[Z1MZuq-8jC8]`，不要单独列表
>
> 强断言规则：每个判断给 `[频道+ID]` 让用户能独立核验；不写"最被低估""最大风险"等情绪化措辞
>
> **不要**加：自批判、已知未知、方法论限制、v1/v2 对照、revision_id 脚手架
>
> 完整结构 A 模板见下文（保留作为复杂任务备选）

## 1. 五大主题聚类

把去重后的所有条目归到 5-8 个核心主题。**不要超过 8 个**——超过说明聚类粒度不够。

| 主题 | 条数 | 代表视频 | 跨平台声量 | 信号强度 |
|------|------|---------|-----------|---------|
| A. [主题名] | N | [视频标题] | [YouTube + B 站 / 仅 B 站] | 高 / 中 / 低 |
| B. [主题名] | N | ... | ... | ... |

**信号强度判定**：
- **高**：跨平台出现 + 一手信源齐全
- **中**：单一平台 + 一手信源可找到
- **低**：单一平台 + 仅有营销/转述

## 4. 数据源 / cron 任务的覆盖盲区（必含，强制段）

| 问题 | 证据 | 影响 |
|------|------|------|
| 视频 URL 不是直链 | 全部为 `bing.com/videos/search?q=...` | 播放量/频道未独立核验 |
| 飞书文档只保留最后写入档 | revision=N, 5067 blocks 仅承载 1 档 | 多档数据无法并置分析 |
| 本地无原始 XML 备份 | `~/.hermes/cron/output/` 最新为 5-30 | 趋势分析无法跨日 |
| YouTube 端 vs B 站端重合度低 | 25 条中仅 1-2 条跨平台 | 中国/国际视角分裂 |
| 时间锚精度低 | "X 天前"而非 ISO 时间戳 | 精确时间序列分析不可行 |

→ **一句话评估**：当前数据是"内容聚合器"，不是"内容分析器"。

---

## 5. 一句话总结

[30-50 字，给出本报告最核心的判断 + 最被低估的信号 + 下一步建议方向]

例：**"2026-06-13 三档视频主流叙事是 X；最被低估的是 Y；同时 Z 暗示了 W 的不可见风险。"**

---

## 使用说明

- **第 0 节（数据范围说明）** + **第 4 节（覆盖盲区）**是**强制段**，不要漏
- **第 3 节（市场共识盲点）**是**用户偏好**，尽量给出 3 个反差
- **第 5 节**给一句话总结，便于转发
- 五维内每个主题一句话落点；某主题在五维中无信号就明说"无显著信号"，不要硬凑
- 表格、emoji、清单是 Feishu Markdown 友好的格式

# 事件流 + 判断流综合分析报告模板（v2）

**适用场景**：用户要求"深度分析""审查""评估"，或报告面向政策制定者/投资者/决策层，或文档条目 > 15 条。

**核心约束**：单一信息流、不重复、强断言必带推演链 + 反方证据、概率评级代替情绪化措辞。

```markdown
# [主题] 综合分析报告

**分析时间**：YYYY-MM-DD HH:MM TZ
**数据源**：飞书 doc token = `<token>`，URL = `https://feishu.cn/docx/<token>`
**v1 → v2 关键修正**：（如适用）反查 YouTube 真实 ID N 条 / 拉 N 个一手公告 / 修正 N 个事实错误

---

| 已外部验证 | N / 总数（X%） | +N（新增验证手段） |
| 仍仅基于标题 | N / 总数（X%） | -N |
| 视频 URL 形态 | （直链 / 搜索页 / 混合） | 提升路径 |
| 时间锚 | （ISO / 相对） | 评估可信度 |

**核心警示**：报告内"代表 X"一列的真实性在 v1 接近 0；v2 中 **N 条真实直链已可独立核验**。

## 写作硬约束（违反任何一条都重写）

- [ ] **每个事件只出现 1-2 次**（不在"清单"和"判断"里重复）
- [ ] **每个强断言配反方证据**（哪怕只是 1 条）
- [ ] **每个数字配时间戳 + 一手来源**（不要"某月""近期"）
- [ ] **概率用"高/中/低置信"**，不用"最大风险""最被低估"
- [ ] **0 节数据范围声明 + 4 节自批判不能省**
- [ ] **不重复 v1 内容**（v2 报告里不要逐字回顾 v1 哪里错）
- [ ] **修复后**说明 v1 → v2 变更，但变更说明不占主报告篇幅（用表格）

## 与 v1 模板（five-dimension-analysis.md）的差异

| 维度 | v1 (五维) | v2 (事件流) |
|---|---|---|
| 组织轴 | 5 个分析视角 | 主体（公司/监管/人）+ 跨主体判断 |
| 同一事件出现次数 | 2-3 次（多维度重复） | 1-2 次（清单 1 次 + 判断 1 次） |
| 强断言位置 | 隐藏在维度叙述里 | 单独成节，配反方 |
| 概率评级 | 缺失 | 强制 |
| 自批判 | "三个反差"伪装 | 显式 4 类（已修正 / 未证实 / 已知未知 / 方法论） |
| 下一步动作 | 一句话带过 | ROI 排序的 5 个具体动作 |
| 适合场景 | 快、概览、面向运营者 | 严、深度、面向决策者 |
